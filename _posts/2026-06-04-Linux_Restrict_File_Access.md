## 1. 실습 환경 구성하기

파일 접근 권한을 제한하는 방법을 연습하기 위해 먼저 `ubuntu` 계정으로 전환하고, 홈 디렉터리에 `myfile.txt` 파일을 만든다.

```bash
su ubuntu
cd ~
touch myfile.txt
```

여기서는 `myfile.txt` 파일을 보안을 위해 누구도 수정하지 못하게 막고 싶은 상황이라고 가정한다.
파일을 수정하지 못하게 하려면 쓰기 권한인 `w`를 제거해야 한다.

---

## 2. 현재 권한 확인하기

파일 권한을 바꾸기 전에 먼저 현재 권한을 확인해야 한다.

```bash
ls -al
```

`ls -al` 결과에서 `myfile.txt`가 아래와 비슷하게 보인다고 가정해보자.

```text
-rw-rw-r-- 1 ubuntu ubuntu 0 Jun  4 10:00 myfile.txt
```

이 출력에서 권한을 나타내는 부분은 맨 앞의 10글자다.

```text
-rw-rw-r--
```

각 부분의 의미는 아래와 같다.

| 위치 | 예시 | 의미 |
|---|---|---|
| 1번째 글자 | `-` | 파일 유형 |
| 2~4번째 글자 | `rw-` | 소유자 권한 |
| 5~7번째 글자 | `rw-` | 소유 그룹 권한 |
| 8~10번째 글자 | `r--` | 그 외 사용자 권한 |

위 예시에서는 소유자가 `ubuntu`이고, 소유 그룹도 `ubuntu`다.
소유자와 소유 그룹은 `rw-` 권한을 가지고 있고, 그 외 사용자는 `r--` 권한만 가지고 있다.

## 3. 파일 권한 문자 이해하기

일반 파일에서 `r`, `w`, `x`는 아래 의미를 가진다.

| 권한 | 의미 | 일반 파일에서의 동작 |
|---|---|---|
| `r` | read | 파일 내용을 읽을 수 있다. |
| `w` | write | 파일 내용을 수정할 수 있다. |
| `x` | execute | 파일을 실행할 수 있다. |

권한은 항상 `r`, `w`, `x` 순서로 표시된다.
해당 권한이 없으면 `-`로 표시된다.

예를 들어 `rw-`는 읽기와 쓰기 권한은 있지만 실행 권한은 없다는 뜻이다.
`r--`는 읽기 권한만 있고 쓰기와 실행 권한은 없다는 뜻이다.

GNU Coreutils 문서는 파일마다 사용자가 접근할 수 있는 종류를 제어하는 file mode bits가 있고, 이 권한을 symbolic form이나 octal number로 표현할 수 있다고 설명한다. [1]

---

## 4. 디렉터리 권한 문자 이해하기

디렉터리에서도 `r`, `w`, `x`를 사용하지만, 일반 파일과 의미가 조금 다르다.

| 권한 | 의미 | 디렉터리에서의 동작 |
|---|---|---|
| `r` | read | 디렉터리 내부 파일 목록을 조회할 수 있다. |
| `w` | write | 디렉터리 내부에 파일을 생성하거나 삭제할 수 있다. |
| `x` | execute/search | 디렉터리 내부로 접근할 수 있다. |

디렉터리에서 `x` 권한은 접근 권한으로 이해하면 된다.
디렉터리 내부에 파일을 생성하거나 삭제하려면 보통 `w` 권한뿐 아니라 `x` 권한도 함께 필요하다.

GNU Coreutils의 numeric mode 설명에서도 execute/search permission이라는 표현을 사용한다. 여기서 `search`는 디렉터리 경로를 탐색할 수 있는 권한을 의미한다. [2]

## 5. 변경할 권한 정하기

`myfile.txt` 파일을 아무도 수정하지 못하게 만들려면 모든 대상에서 쓰기 권한인 `w`를 제거해야 한다.

대상은 아래 세 가지로 나뉜다.

| 대상 | 의미 |
|---|---|
| 소유자 | 파일을 소유한 사용자 |
| 소유 그룹 | 파일을 소유한 그룹 |
| 그 외 사용자 | 소유자도 아니고 소유 그룹에도 속하지 않은 사용자 |

모든 대상이 파일을 읽을 수는 있지만 수정할 수는 없게 만들려면 권한을 아래처럼 설정하면 된다.

```text
r--r--r--
```

이 권한은 소유자, 소유 그룹, 그 외 사용자 모두에게 읽기 권한만 부여한다.
쓰기 권한인 `w`가 없기 때문에 파일을 수정할 수 없다.

---

## 6. 권한 문자를 숫자로 바꾸기

`chmod` 숫자 권한에서는 각 권한을 아래 숫자로 계산한다.

| 권한 | 숫자 |
|---|---|
| `r` | `4` |
| `w` | `2` |
| `x` | `1` |

`r--`는 읽기 권한만 있으므로 아래처럼 계산한다.

```text
r-- = 4 + 0 + 0 = 4
```

소유자, 소유 그룹, 그 외 사용자에게 모두 `r--` 권한을 부여하려면 숫자 세 자리가 모두 `4`가 된다.

```text
r-- r-- r--
 4   4   4
```

따라서 `myfile.txt`를 누구도 수정하지 못하게 제한하는 명령어는 아래와 같다.

```bash
chmod 444 myfile.txt
```

GNU Coreutils 문서는 numeric mode에서 소유자, 그룹, 그 외 사용자 각각의 읽기, 쓰기, 실행 권한을 octal digit으로 표현한다고 설명한다. [2]

## 7. 변경된 권한 확인하기

권한을 변경한 뒤에는 다시 `ls -al`로 결과를 확인한다.

```bash
ls -al
```

정상적으로 적용되었다면 `myfile.txt`의 권한이 아래처럼 보인다.

```text
-r--r--r-- 1 ubuntu ubuntu 0 Jun  4 10:00 myfile.txt
```

앞의 10글자를 보면 `w`가 모두 사라진 것을 확인할 수 있다.

```text
-r--r--r--
```

이제 소유자, 소유 그룹, 그 외 사용자 모두 파일을 읽을 수는 있지만 수정할 수는 없다.

---

## 8. 수정이 막히는지 테스트하기

파일을 열어서 실제로 수정이 막히는지 확인해보자.

```bash
vi myfile.txt
```

`vim`으로 파일을 열면 읽기 전용 상태를 의미하는 `[readonly]` 표시가 보일 수 있다.
`i`를 눌러 INSERT 모드로 전환하고 내용을 입력한 뒤 저장을 시도해보면 저장이 되지 않는다.

쓰기 권한이 없기 때문에 일반적인 저장 명령으로는 파일을 수정할 수 없다.
즉, `chmod 444 myfile.txt`를 통해 의도한 대로 쓰기 권한이 제한된 것이다.

---

## 핵심 정리

파일을 아무도 수정하지 못하게 하려면 쓰기 권한인 `w`를 제거해야 한다.
일반 파일에서 `w`는 파일 내용을 수정할 수 있는 권한이다.

`r--`는 숫자로 `4`다.
소유자, 소유 그룹, 그 외 사용자 모두에게 읽기 권한만 부여하려면 `444`를 사용한다.

```bash
chmod 444 myfile.txt
```

권한을 변경한 뒤에는 반드시 `ls -al`로 결과를 확인해야 한다.
권한이 `-r--r--r--`로 바뀌었다면 세 대상 모두에게 읽기 권한만 남아 있고, 쓰기 권한은 제거된 상태다.
