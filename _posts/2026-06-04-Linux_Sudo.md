## 1. 권한이 없어도 접근할 수 있는 경우

일반 사용자 권한으로는 접근할 수 없는 파일이나 디렉터리가 있다.
예를 들어 `ubuntu` 계정으로 로그인한 뒤 `/var/log` 디렉터리를 확인해보자.

```bash
su ubuntu
cd /var/log
ls -al
```

`/var/log` 내부에 `chrony`라는 디렉터리가 있고, 권한이 아래와 비슷하게 설정되어 있다고 가정한다.

```text
drwxr-x--- 2 _chrony _chrony 4096 Jun  4 10:00 chrony
```

이 권한을 해석하면 아래와 같다.

| 대상 | 권한 | 의미 |
|---|---|---|
| 소유자 `_chrony` | `rwx` | 읽기, 쓰기, 접근 가능 |
| 소유 그룹 `_chrony` | `r-x` | 읽기, 접근 가능 |
| 그 외 사용자 | `---` | 아무 권한 없음 |

현재 사용자가 `ubuntu`이고 `_chrony` 사용자나 `_chrony` 그룹에 속하지 않는다면, `chrony` 디렉터리에 접근할 권한이 없다.

실제로 아래 명령어를 실행하면 `Permission denied`가 발생할 수 있다.

```bash
ls chrony
```

## 2. root 사용자로 접근하기

이번에는 `root` 사용자로 전환한 뒤 같은 명령어를 실행해보자.

```bash
sudo su
ls chrony
```

이번에는 `Permission denied`가 발생하지 않을 수 있다.
명령어를 실행했는데 아무것도 출력되지 않는다면, 권한 문제가 아니라 `chrony` 디렉터리 내부에 출력할 파일이 없기 때문일 수 있다.

`root` 사용자가 `_chrony` 사용자가 아닌데도 접근할 수 있는 이유는 `root`가 슈퍼 사용자이기 때문이다.
슈퍼 사용자는 시스템 관리 권한을 가진 계정이며, 일반 사용자에게 적용되는 파일 접근 제한을 우회해 관리 작업을 수행할 수 있다.

다만 슈퍼 사용자 권한은 강력하기 때문에 실수로 시스템 파일을 수정하거나 삭제하면 시스템에 문제가 생길 수 있다.

---

## 3. `sudo`란?

`sudo`는 허용된 사용자가 특정 명령어를 다른 사용자 권한으로 실행할 수 있게 해주는 명령어다.
일반적으로 별도 사용자를 지정하지 않으면 `root` 권한으로 명령어를 실행한다.

Sudo 공식 매뉴얼은 `sudo`를 다른 사용자로 명령을 실행하는 도구라고 설명한다. 또한 기본 보안 정책은 `sudoers`이며, 이 설정은 `/etc/sudoers` 파일이나 LDAP를 통해 구성된다고 설명한다. [1]

기본 형태는 아래와 같다.

```bash
sudo [실행하고자 하는 명령어]
```

예를 들어 일반 사용자로는 접근할 수 없는 `chrony` 디렉터리를 확인해야 한다면 아래처럼 실행할 수 있다.

```bash
sudo ls chrony
```

이 명령어는 `ls chrony`를 현재 사용자 권한이 아니라 관리자 권한으로 실행한다.

## 4. `sudo`로 권한 없는 파일 읽기

`/etc` 디렉터리에는 시스템 설정 파일들이 들어 있다.
그중 `sudoers` 파일은 누가 어떤 명령을 `sudo`로 실행할 수 있는지에 대한 설정을 담는 파일이다.

먼저 `/etc` 디렉터리로 이동한 뒤 `sudoers` 파일의 권한을 확인한다.

```bash
cd /etc
ls -l sudoers
```

환경에 따라 출력은 다를 수 있지만, 일반적으로 `root`가 소유하고 일반 사용자는 직접 수정할 수 없도록 제한되어 있다.

일반 사용자 권한으로 파일 내용을 출력하려고 하면 권한 오류가 발생할 수 있다.

```bash
cat sudoers
```

이때 `sudo`를 붙이면 관리자 권한으로 `cat` 명령어를 실행할 수 있다.

```bash
sudo cat sudoers
```

Sudo 공식 매뉴얼은 `sudoers` 정책이 기본 정책이며, 사용자가 허용된 명령만 실행할 수 있도록 제어한다고 설명한다. [1]
따라서 `sudo`는 아무 사용자에게나 무조건 관리자 권한을 주는 명령어가 아니라, 설정상 허용된 사용자에게만 동작한다.

## 5. root 사용자 전환과 `sudo`의 차이

권한이 부족할 때 해결하는 방식은 크게 두 가지가 있다.

| 방식 | 예시 | 특징 |
|---|---|---|
| root 사용자로 전환 | `sudo su` | 이후 명령을 계속 root 사용자로 실행 |
| 필요한 명령에만 `sudo` 사용 | `sudo ls chrony` | 해당 명령만 관리자 권한으로 실행 |

`sudo su`를 사용하면 `root` 사용자 셸로 전환된다.
이후에는 명령어마다 `sudo`를 붙이지 않아도 된다.

```bash
sudo su
```

반면 `sudo [명령어]` 방식은 필요한 명령어 앞에만 `sudo`를 붙인다.

```bash
sudo ls chrony
sudo cat /etc/sudoers
```

두 방식 모두 권한 문제를 해결할 수 있지만, 실습이나 실제 운영 환경에서는 필요한 명령어에만 `sudo`를 붙이는 방식이 더 안전하다.
`root` 셸로 오래 작업하면 실수로 중요한 시스템 파일을 변경하거나 삭제할 가능성이 커지기 때문이다.

## 6. 언제 `sudo`를 사용할까?

`sudo`는 현재 사용자 권한으로는 할 수 없지만, 시스템 관리상 꼭 필요한 작업을 수행할 때 사용한다.

예를 들면 아래와 같은 상황이다.

- 시스템 설정 파일을 읽거나 수정해야 할 때
- 패키지를 설치하거나 업데이트해야 할 때
- 시스템 서비스 상태를 확인하거나 재시작해야 할 때
- 일반 사용자 권한으로 접근할 수 없는 로그 파일을 확인해야 할 때

하지만 `Permission denied`가 발생했다고 해서 항상 바로 `sudo`를 붙이는 것은 좋지 않다.
먼저 현재 사용자, 파일 소유자, 소유 그룹, 권한을 확인한 뒤 정말 관리자 권한이 필요한 상황인지 판단해야 한다.

```bash
whoami
ls -al
groups
```

권한 구조를 확인한 뒤에도 관리자 권한이 필요한 작업이라면 그때 `sudo`를 사용한다.

## 정리

`root`는 시스템의 관리자 역할을 하는 슈퍼 사용자다.
일반 사용자에게 허용되지 않은 파일이나 디렉터리에도 관리 목적으로 접근할 수 있다.

`sudo`는 허용된 사용자가 특정 명령어를 관리자 권한으로 실행할 때 사용하는 명령어다.
기본 형태는 아래와 같다.

```bash
sudo [실행하고자 하는 명령어]
```

권한 문제를 해결할 때는 `sudo su`로 계속 root 사용자로 작업하기보다, 필요한 명령어에만 `sudo`를 붙이는 편이 더 안전하다.
또한 `sudo`를 사용하기 전에는 먼저 `whoami`, `ls -al`, `groups`로 현재 사용자와 권한 상태를 확인하는 습관을 들이는 것이 좋다.
