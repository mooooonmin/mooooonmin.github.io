## 1. 리눅스란?

리눅스를 처음 검색하면 보통 "유닉스 계열의 오픈 소스 운영체제" 또는 "리눅스 커널 기반 운영체제"처럼 설명되어 있다.

이 설명은 틀린 말은 아니지만, 처음 공부하는 입장에서는 어렵게 느껴질 수 있다.
처음에는 다음처럼 이해해도 충분하다.

```text
Linux는 Windows나 macOS처럼 컴퓨터를 사용할 수 있게 해주는 운영체제 계열이다.
```

운영체제는 컴퓨터 하드웨어와 사용자가 실행하는 프로그램 사이에서 중간 역할을 하는 소프트웨어다.
Red Hat 문서도 운영체제가 CPU, 메모리, 저장소 같은 시스템 자원을 관리한다고 설명한다. [1]

즉, 운영체제가 있어야 사용자는 파일을 만들고, 프로그램을 실행하고, 네트워크에 연결하고, 하드웨어 자원을 사용할 수 있다.

---

## 2. 운영체제를 쉽게 이해하기

컴퓨터에는 CPU, 메모리, 디스크, 네트워크 장치 같은 하드웨어가 있다.
하지만 사용자가 이 하드웨어를 직접 제어하면서 프로그램을 실행하기는 어렵다.

운영체제는 이 과정을 대신 관리한다.

| 역할 | 설명 |
|---|---|
| 프로그램 실행 | 사용자가 실행한 프로그램을 컴퓨터에서 동작하게 한다. |
| 파일 관리 | 파일을 저장하고 읽을 수 있게 한다. |
| 하드웨어 관리 | CPU, 메모리, 디스크 같은 자원을 나누어 사용하게 한다. |
| 사용자 인터페이스 제공 | 그래픽 화면 또는 명령어 입력 방식으로 컴퓨터를 조작하게 한다. |

Windows, macOS, Linux는 모두 운영체제로 이해할 수 있다.

---

## 3. Linux와 Linux 커널

정확히 말하면 Linux는 원래 **커널(kernel)** 이름이다.

커널은 운영체제의 핵심 부분이다.
하드웨어 자원을 관리하고, 프로그램이 하드웨어를 사용할 수 있도록 연결해준다.

Linux Kernel 공식 문서는 Linux 커널 자체에 대한 문서를 제공하고 있으며, Red Hat은 Linux 기반 운영체제가 Linux 커널과 여러 소프트웨어 패키지로 구성된다고 설명한다. [1][2]

따라서 실무에서 "Linux를 설치한다"라고 말할 때는 보통 Linux 커널만 설치한다는 뜻이 아니다.
Ubuntu, Rocky Linux, Debian, Fedora 같은 **Linux 배포판(distribution)**을 설치한다는 뜻에 가깝다.

정리하면 다음과 같다.

| 구분 | 의미 |
|---|---|
| Linux 커널 | 운영체제의 핵심 부분 |
| Linux 배포판 | Linux 커널에 기본 프로그램, 라이브러리, 패키지 관리 도구 등을 묶어 배포하는 운영체제 |

---

## 4. 리눅스는 왜 검은 화면에서 명령어를 입력할까?

리눅스에도 그래픽 화면이 있다.
Ubuntu Desktop처럼 마우스로 클릭하면서 사용할 수 있는 Linux 배포판도 있다.

하지만 서버 환경에서는 그래픽 화면보다 명령어 기반 화면을 많이 사용한다.
서버는 사용자가 직접 화면을 보면서 작업하기보다, 원격 접속으로 필요한 명령을 실행하고 서비스를 운영하는 경우가 많기 때문이다.

그래서 Linux를 공부하다 보면 다음과 같은 검은 화면을 자주 보게 된다.

```bash
ls
cd /home
pwd
mkdir test
```

처음에는 낯설지만, 명령어 방식은 서버를 빠르고 정확하게 관리하는 데 유리하다.
Docker, Kubernetes, 클라우드 서버를 다룰 때도 Linux 명령어를 자주 사용한다.

---

## 5. 처음에는 어떻게 이해하면 될까?

처음 공부할 때는 Linux를 너무 어렵게 정의하려고 하지 않아도 된다.

먼저 다음 문장만 기억하면 된다.

```text
Linux는 Windows나 macOS 같은 운영체제 계열이고,
서버 환경에서 명령어로 많이 다루는 운영체제다.
```

그다음 공부가 조금 익숙해지면 다음 단계로 이해를 확장하면 된다.

1. Linux는 원래 커널 이름이다.
2. 실제로 설치해서 쓰는 것은 보통 Linux 배포판이다.
3. Linux 배포판에는 커널, 기본 명령어, 라이브러리, 패키지 관리 도구가 함께 들어 있다.
4. 서버 환경에서는 GUI보다 CLI를 많이 사용한다.

---

## 핵심 정리

> Linux는 처음에는 Windows나 macOS 같은 운영체제 계열로 이해하면 된다.
>
> 운영체제는 컴퓨터 하드웨어와 프로그램 사이에서 자원을 관리하는 소프트웨어다.
>
> 정확히 말하면 Linux는 커널이고, Ubuntu나 Debian 같은 배포판이 우리가 설치해서 쓰는 Linux 운영체제에 가깝다.
>
> 서버 환경에서는 Linux를 그래픽 화면보다 명령어로 다루는 경우가 많다.
