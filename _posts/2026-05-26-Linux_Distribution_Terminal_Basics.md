## 1. 리눅스 배포판이란?

Linux는 오픈 소스로 공개되어 있기 때문에 여러 사람과 기업이 목적에 맞게 구성해서 사용할 수 있다.

이렇게 Linux 커널에 기본 프로그램, 라이브러리, 패키지 관리 도구, 설정 도구 등을 함께 묶어 배포하는 운영체제를 **Linux 배포판**이라고 부른다.

대표적인 Linux 배포판에는 다음과 같은 것들이 있다.

| 배포판 | 특징 |
|---|---|
| Ubuntu | 입문자와 서버 환경 모두에서 많이 사용되는 Debian 기반 배포판 |
| Debian | 안정성과 자유 소프트웨어 철학을 중요하게 다루는 배포판 |
| Fedora | 비교적 새로운 기술을 빠르게 반영하는 배포판 |
| CentOS Stream | Red Hat Enterprise Linux와 연결된 서버 중심 배포판 |

Debian 공식 문서는 Debian을 자유 운영체제로 설명하며, 운영체제의 핵심에는 커널이 있고 그 위에 기본 도구와 소프트웨어가 쌓인다고 설명한다. [1]
Ubuntu 공식 사이트도 Ubuntu를 오픈 소스 운영체제로 설명하고, 데스크톱과 서버 환경에서 사용할 수 있는 배포판으로 제공한다. [2]

즉, Linux 배포판은 다음처럼 이해하면 된다.

```text
Linux 배포판 = Linux 커널 + 기본 프로그램 + 패키지 관리 도구 + 배포판별 설정
```

---

## 2. 처음에는 어떤 배포판을 쓰면 좋을까?

Linux에 막 입문하는 단계라면 **Ubuntu**를 사용하는 것이 무난하다.

Ubuntu는 공식 문서와 커뮤니티 자료가 많고, 데스크톱과 서버 환경 모두에서 접하기 쉽다.
Ubuntu 공식 사이트도 Ubuntu Desktop을 오픈 소스 운영체제로 소개하고, 개발자용 도구와 일반 데스크톱 사용에 필요한 프로그램을 제공한다고 설명한다. [2]

처음 공부할 때 중요한 것은 배포판별 장단점을 모두 외우는 것이 아니다.
먼저 하나의 배포판을 골라 명령어, 파일 구조, 패키지 설치 방식, 서버 접속 방식에 익숙해지는 것이 더 중요하다.

추천 흐름은 다음과 같다.

1. 처음에는 Ubuntu로 시작한다.
2. 기본 명령어와 터미널 사용법을 익힌다.
3. 서버 환경에 익숙해진다.
4. 필요해지면 Debian, Fedora, Rocky Linux, CentOS Stream 같은 다른 배포판의 차이를 비교한다.

## 3. 터미널 첫 화면 해석하기

Linux 서버에 접속하면 보통 다음과 비슷한 화면을 보게 된다.

```bash
ubuntu@ip-172-31-24-185:~$
```

처음 보면 복잡해 보이지만, 나누어 보면 어렵지 않다.

| 부분 | 의미 |
|---|---|
| `ubuntu` | 현재 로그인한 사용자 이름 |
| `@` | 사용자 이름과 호스트 이름을 구분하는 문자 |
| `ip-172-31-24-185` | 호스트 이름, 즉 컴퓨터 이름 |
| `:` | 호스트 이름과 현재 경로를 구분하는 문자 |
| `~` | 현재 사용자의 홈 디렉터리 |
| `$` | 일반 사용자의 명령 입력 대기 표시 |

GNU Bash 매뉴얼에 따르면 Bash 프롬프트에서는 `\u`가 현재 사용자 이름, `\h`가 호스트 이름, `\w`가 현재 작업 디렉터리를 의미한다. 또한 `\$`는 사용자가 root이면 `#`, 그렇지 않으면 `$`로 표시된다. [3]

따라서 위 프롬프트는 다음처럼 읽을 수 있다.

```text
ubuntu 사용자로
ip-172-31-24-185 라는 컴퓨터에 접속해 있고
현재 위치는 홈 디렉터리이며
명령어 입력을 기다리고 있다.
```

## 4. 기본 조작법 익히기

터미널은 마우스로 버튼을 누르는 방식보다, 키보드로 명령어를 입력하는 방식에 가깝다.
처음에는 아래 조작만 익혀도 충분하다.

### 아무 명령어나 입력하고 Enter 누르기

예를 들어 다음처럼 입력하고 Enter를 누른다.

```bash
pwd
```

`pwd`는 현재 위치를 출력하는 명령어다.

### Backspace로 입력 지우기

명령어를 잘못 입력했다면 `Backspace`로 지우면 된다.
일반 텍스트 입력과 비슷하게 생각하면 된다.

### 방향키로 커서 이동하기

입력 중인 명령어 안에서 커서를 좌우로 움직이고 싶다면 방향키를 사용한다.

```text
왼쪽 이동: ←
오른쪽 이동: →
```

명령어가 길어질수록 커서를 이동해서 중간만 고치는 일이 많아진다.

---

## 5. 단어 단위로 커서 이동하기

긴 명령어를 입력할 때는 글자 하나씩 이동하는 것보다 단어 단위로 이동하는 것이 편하다.

환경에 따라 단축키는 다를 수 있다.

| 환경 | 단어 단위 이동 |
|---|---|
| Windows 터미널 계열 | `Ctrl + ←`, `Ctrl + →` 또는 `Alt + ←`, `Alt + →` |
| macOS 터미널 계열 | `Option + ←`, `Option + →` |

터미널 프로그램, 셸, 키보드 설정에 따라 동작이 다를 수 있으므로 실제 환경에서 직접 눌러 확인하는 것이 좋다.

## 6. 입력을 취소하고 싶을 때

명령어를 길게 입력하다가 처음부터 다시 쓰고 싶다면 `Ctrl + C`를 누른다.

```text
Ctrl + C
```

`Ctrl + C`는 현재 입력 중인 명령이나 실행 중인 프로그램을 중단할 때 자주 사용한다.
터미널에서 `Ctrl + C`를 누르면 화면에 `^C`가 표시될 수 있다.

여기서 `^`는 `Ctrl` 키를 의미하는 표시로 이해하면 된다.

예를 들어 입력 중인 명령을 취소하면 다음처럼 보일 수 있다.

```bash
ubuntu@ip-172-31-24-185:~$ abcdefg^C
ubuntu@ip-172-31-24-185:~$
```

## 7. 터미널 화면 지우기

터미널 화면이 복잡해졌다면 `clear` 명령어로 화면을 정리할 수 있다.

```bash
clear
```

`clear`는 기존 명령 실행 결과를 지우고 화면을 깨끗하게 보여준다.
명령 이력 자체가 삭제되는 것은 아니고, 화면 표시만 정리된다고 이해하면 된다.

---

## 8. 복사와 붙여넣기

터미널에서는 일반 문서 편집기와 복사/붙여넣기 단축키가 다를 수 있다.

| 환경 | 복사 | 붙여넣기 |
|---|---|---|
| Windows 터미널 계열 | `Ctrl + Insert` | `Shift + Insert` |
| macOS 터미널 계열 | `Command + C` | `Command + V` |

Windows 환경의 일부 터미널에서는 `Ctrl + C`가 복사가 아니라 명령 중단으로 동작할 수 있다.
그래서 터미널에서 복사/붙여넣기가 잘 되지 않는다면 `Ctrl + Insert`, `Shift + Insert`를 먼저 확인해보는 것이 좋다.

## 핵심 정리

> Linux 배포판은 Linux 커널에 여러 프로그램과 도구를 묶어 사용할 수 있게 만든 운영체제다.
>
> 처음 Linux를 공부한다면 Ubuntu로 시작하는 것이 무난하다.
>
> 터미널 프롬프트의 `ubuntu@host:~$`는 사용자, 컴퓨터 이름, 현재 위치, 명령 입력 대기 상태를 보여준다.
>
> 처음에는 Enter, Backspace, 방향키, `Ctrl + C`, `clear`, 복사/붙여넣기만 익혀도 터미널 사용을 시작할 수 있다.
