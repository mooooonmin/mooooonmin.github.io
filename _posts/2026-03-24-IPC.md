## 1. IPC 개념

Process는 서로 **독립적인 주소 공간을 가지는 실행 단위** 의미.
따라서 한 Process가 다른 Process의 Memory 영역을 직접 참조하는 것은 **불가능한 구조**.
이 문제를 해결하기 위한 메커니즘이 **IPC (Inter Process Communication)**.
IPC는 Process 간 데이터를 주고받기 위한 **통신 방식** 의미.

---

## 2. IPC 필요성

Process 구조의 특징

- Memory 공간 독립 구조
- 직접 데이터 공유 불가능

결과

- Process 간 데이터 교환 불가능

해결 방법

- 운영체제에서 IPC 제공 구조

---

## 3. IPC 방식

IPC는 두 가지 모델로 구분.

- **공유 메모리 (Shared Memory)**
- **메시지 전달 (Message Passing)**

---

## 4. 공유 메모리 (Shared Memory)

여러 Process가 **하나의 Memory 영역을 공유하는 방식**.

### 동작 구조

1. Process가 Kernel에 공유 메모리 요청
2. Kernel이 Memory 영역 할당
3. Process가 해당 영역 직접 접근

### 특징

- Kernel 개입 최소 구조
- 일반 Memory 접근 방식 사용
- 높은 성능 구조

### 장점

- 매우 빠른 통신 속도
- Kernel 호출 최소화

### 단점

- 동기화 문제 발생 가능
- Race Condition 발생 가능
- 직접 동기화 처리 필요

---

## 5. 메시지 전달 (Message Passing)

Kernel을 통해 데이터를 주고받는 방식.

### 동작 구조

- send(message)
- receive(message)

Process → Kernel → Process 구조

### 특징

- Kernel이 통신 중재
- Memory 공유 없음

### 장점

- 동기화 문제 없음
- 구현 단순 구조
- 안정성 높음

### 단점

- Kernel 개입으로 인한 속도 저하
- 통신 오버헤드 존재

---

## 6. IPC 방식 비교

| 구분 | 공유 메모리 | 메시지 전달 |
|---|---|---|
| 속도 | 빠름 | 느림 |
| Kernel 개입 | 없음 (초기만) | 항상 존재 |
| 동기화 | 필요 | 불필요 |
| 안정성 | 낮음 | 높음 |
| 구현 난이도 | 높음 | 낮음 |

---

## 7. IPC 예시

### 공유 메모리 방식

- Shared Memory
- POSIX Shared Memory

### 메시지 전달 방식

- Pipe
- Socket
- Message Queue

## 핵심 정리

>
> 공유 메모리
