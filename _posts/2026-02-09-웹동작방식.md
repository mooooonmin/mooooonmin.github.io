## 1. 핵심 요약 (Quick View)
1. **URL 입력:** 사용자가 브라우저 주소창에 `www.google.com` 입력함.
2. **DNS 조회:** 도메인 이름을 실제 IP 주소로 변환함.
3. **연결 설정:** TCP 3-Way Handshake를 통해 서버와 연결을 맺음.
4. **HTTP 요청/응답:** 요청 메시지를 패킷에 담아 보내고, 서버로부터 응답을 받음.
5. **브라우저 렌더링:** 받은 데이터를 바탕으로 화면을 그려줌.

---

## 2. 웹 동작 상세 과정

### Step 1. 주소창 입력 및 DNS Lookup
* 브라우저에 URL을 치면, 가장 먼저 해당 도메인의 **IP 주소**를 찾아야 함.
* **과정:** 브라우저 캐시 → OS 캐시 → 라우터 캐시 → ISP(통신사) DNS 서버 순으로 뒤져서 IP를 알아냄.

### Step 2. TCP 연결 및 TLS 핸드쉐이크
* IP 주소를 알았으니 이제 서버랑 통로를 뚫어야 함.
* **TCP 3-Way Handshake:** `SYN` → `SYN-ACK` → `ACK` 과정을 통해 연결을 확정함.
* **HTTPS 보안:** 요즘은 대부분 HTTPS를 쓰니까, 비대칭키 암호화를 활용한 **TLS 핸드쉐이크** 과정도 여기서 함께 일어남.



### Step 3. HTTP 요청 메시지 생성 및 전송
* 연결된 통로로 **HTTP Request 메시지**를 보냄.
* 이 메시지는 다시 **TCP/IP 패킷**으로 감싸짐(Encapsulation).
* 헤더에는 출발지 IP/Port, 목적지 IP/Port 같은 정보가 붙어서 랜선을 타고 전기로 변해 날아감.

### Step 4. 서버의 처리 및 응답 전송
* 구글 서버에 도착한 패킷은 다시 언패킹(Unpacking)되어 메시지로 복원됨.
* 서버 프로세스가 요청을 읽고 필요한 데이터(검색창 HTML 등)를 담아 **HTTP Response 메시지**를 만듦.
* 왔던 방식 그대로 다시 클라이언트 IP로 패킷을 쏴줌.

### Step 5. 브라우저 렌더링
* 도착한 응답 메시지 바디에 담긴 HTML, CSS, JS 데이터를 브라우저가 읽음.
* **렌더링 엔진**이 열일해서 우리 눈에 보이는 구글 검색창 화면을 모니터에 그려줌.
