# ✅ Ch1. 시스템 보안

## 1) `/etc/passwd` 구성 — **계패유지설홈쉘**

- 계정, 패스워드표시, UID, GID, 설명, 홈디렉터리, 로그인쉘
- 패스워드 `x` → 실제 해시는 `/etc/shadow`
- root UID = 0
- `/usr/bin/passwd`는 **비밀번호 변경 프로그램**(DB 파일 아님)

## 2) `/etc/shadow` 구성 — **유엔마민맥 경인만**

- 유저계정, 암호화 pw, 마지막 변경일(1970 기준 day로 표현되기도), 최소, 최대, 경고, inactive, 만료일

## 3) 파일 타입/권한/명령

- 파일 타입:  일반파일 / `d` 디렉터리 / `l` 심볼릭링크
- 권한: r(4) 읽기 / w(2) 쓰기 / x(1) 실행
- chmod, chown, chgrp
- umask: 기본권한에서 마스킹
    - 기본 파일 666 / 디렉터리 777
    - 예) **umask 027 → 파일 640, 디렉 750** (기출 단골)

## 4) inode / 링크 (시험 단골)

- inode에 **없는 것 = 파일명**
- 하드링크: 같은 inode 공유 → 수정 반영 O / 원본 이름 삭제해도 다른 링크는 남음
- 심볼릭링크: 경로 참조 → 원본 삭제 시 깨짐

## 5) find(권한 탐색)

- setuid 찾기: `find / -perm -4000 -type f`
- `perm -222` : 해당 비트들이 **모두** 설정(AND)
    
    `/222` : 하나라도(OR) (함정 주의)
    

## 6) crontab — **분시일월요(명령)**

- `0 0 1 * * cmd` : 매달 1일 0시
- allow/deny로 사용자 제한

## 7) 특수권한 — **특유그기(4/2/1)**

- setuid(4000): 실행 시 파일 **소유자 권한**
- setgid(2000): 실행 시 **그룹 권한**, 디렉은 그룹 상속
- sticky(1000): (디렉터리에서) **삭제/이름변경은 소유자/디렉소유자/root만**
    
    (`/tmp` = drwxrwxrwt)
    

## 8) xinetd / TCP Wrapper

- xinetd: service/disable/socket_type/wait/user/server + only_from/no_access/cps/per_source/access_time/instances
- TCP Wrapper: `/etc/hosts.allow`, `/etc/hosts.deny` (개념 분리해서 암기)

## 9) PAM — **어오패세**

- auth(인증), account(계정 정책), password(변경), session(세션 처리)
- control flag: requisite/required/sufficient/optional

## 10) 시스템 로그(교정 완료: 매우 중요)

- **utmp**: 현재 로그인(`who`)
- **wtmp**: 로그인/로그아웃 이력(`last`)
- **btmp**: 실패 이력(`lastb`)
- **lastlog**: 사용자별 마지막 로그인
- sulog: su 기록
- acct/pacct: 명령/프로세스 회계
- xferlog: FTP 전송 로그
- 로그 원칙: **중앙 수집/원격 백업이 바람직**(“필요없다”는 보기 함정)

## 11) 패스워드 정책/세션

- PASS_MIN_DAYS / PASS_MAX_DAYS / PASS_MIN_LEN 등
- 정책 전환: pwconv/pwunconv
- TMOUT: 쉘 자동 로그아웃

## 12) 프로세스

- 좀비 프로세스: `Z`
    
    `ps aux | grep Z` / `ps -el | grep Z`
    

## 13) Windows 그룹/공유(기출)

- 기본 그룹: Administrators / Account Operators / Backup Operators / Power Users(관리자 상위 아님)
- 기본 공유: IPC$, ADMIN$, C$ (관리자 접근)

# ✅ Ch2. 네트워크 보안

## 1) TCP/IP 계층(“네트워크 접속계층” 정상 용어)

- 응용 / 전송 / 인터넷 / **네트워크 접속(Network Access)**
- 함정: 프로토콜 계층 바꿔치기(TCP/UDP/IP/ICMP/ARP)

## 2) ARP Spoofing(2계층)

- ARP: IP→MAC
- 공격: ARP spoofing/poisoning(MITM)
- 방어: 정적 ARP, ARP watch, DHCP snooping+DAI

## 3) IP Spoofing(3계층)

- 방어: ingress/egress filtering(BCP38), uRPF, ACL
- rhosts/hosts.equiv 권한 엄격(600 이하 등) 출제됨

## 4) ICMP Redirect(3계층)

- 방어: `accept_redirects=0`
- 라우팅 테이블: `netstat -rn`

## 5) 네트워크 명령

- ping: ICMP echo
- traceroute: TTL 증가 + **ICMP Time Exceeded**(핵심)
- ifconfig promisc: **무차별모드 활성**(해제 아님)

## 6) 포트 스캐닝(응답 패턴 단골)

- TCP connect: SYN → SYN/ACK → ACK(+RST)
- SYN scan(half-open): SYN → SYN/ACK → RST
- FIN/NULL/XMAS: open=무응답 / closed=RST
- ACK scan: 방화벽 정책 확인(필터링 여부)
- UDP scan: closed=ICMP unreachable / open=무응답(많음)

## 7) Nmap 옵션(빈출)

- `sS/-sT/-sU/-sA`, `sV`, `O`, `p`
- `Pn` ping skip
- `f` 단편화
- `D` decoy
- “스텔스 아닌 것” → connect(-sT)로 자주 출제

## 8) DoS/DDoS/DRDoS

- DoS: 자원 고갈(특정 서비스 반복 호출 등)
- DRDoS: 반사/증폭(UDP 기반: DNS/NTP/SSDP/CLDAP 등)
- Smurf: ICMP directed broadcast → `no ip directed-broadcast`

## 9) SYN Flood 대응

- SYN cookies / backlog 튜닝 / rate limit / 앞단 스크러빙 등

## 10) VPN/IPSec/SSL

- IPSec: L3
- AH: 인증/무결성(암호화 X)
- ESP: 암호화 + (옵션) 인증/무결성
- IPSec: **패킷 단위**, SSL VPN: 세션/메시지 단위

## 11) 무선

- IPv4 멀티캐스트 존재(Class D)
- WPA=TKIP, WPA2=CCMP(AES) (TKIP 문제 단골)

# ✅ Ch3. 응용프로그램 보안

## 1) DNS (정답 포트)

- DNS: **53/UDP, 53/TCP**
- 존전송(AXFR): TCP
- 재귀 vs 반복(권한) 구분(시나리오 함정)
- `dig -x` 역조회(PTR)
- DNSSEC: 스푸핑/캐시포이즈 방어

## 2) HTTP / 쿠키

- httponly / secure / SameSite(추가로 자주 나옴)

## 3) FTP / TFTP / SFTP / FTPS

- FTP 제어: 21/TCP
- 데이터: Active=20/TCP / Passive=서버 **1024+ 임의 포트**
- TFTP: UDP/69 (취약)
- SFTP: SSH 기반, 22/TCP (SSL/TLS 쓰는 건 FTPS)
- FTP bounce: PORT 악용(서버가 제3자 연결)

## 4) SNMP

- 161/162
- v3 권장(인증/암호화), 커뮤니티 스트링 변경

## 5) SQLi / XSS / CSRF (빈출)

- SQLi: `1=1`, union/error/blind
    - 대응: **Prepared Statement**
- XSS: Stored/Reflected/DOM
    - 대응: 출력 인코딩 + CSP
- CSRF: 토큰/Origin-Referer/SameSite
    - 함정: “HTML 인코딩”을 CSRF 대응으로 제시(오답)

## 6) 업로드/다운로드/경로/세션

- 업로드: 확장자/콘텐츠 검사, 저장경로 격리, 실행권한 제거
- 디렉 리스팅: Options Indexes 제거
- 세션: timeout, cookie_secure/httponly, gc_maxlifetime

## 7) 이메일 보안

- PGP vs S/MIME
    - S/MIME: **X.509 인증서**
- SMTP/POP3/IMAP 포트 및 보안버전(465/587, 995, 993 등) 출제 가능

# ✅ Ch4. 보안장비 운용

## 1) IDS/IPS/WAF/UTM/NAC

- IDS: 탐지/경보
- IPS: **탐지+차단(in-line)**
- WAF: L7(응용) 기반, ModSecurity 등
- UTM: 통합(방화벽+IDS/IPS+VPN 등)
- NAC: 단말 접근제어

## 2) Snort 룰

- action: alert/log/drop/reject
- option: msg/sid/content/offset/depth/distance/within/nocase
- threshold/limit/both(출제 단골)

## 3) iptables

- 체인: INPUT/OUTPUT/FORWARD
- 정책: ACCEPT/DROP/REJECT/LOG
- state: ESTABLISHED/RELATED 활용

## 4) 무결성/루트킷/스캐너

- Tripwire: 무결성 점검
- chkrootkit: 루트킷 점검
- Nessus/SAINT/SATAN류: 취약점 점검(고전 기출)

## 5) APT/킬체인

- Kill chain 7단계: Recon→Weaponization→Delivery→Exploitation→Installation→C2→Actions
- lateral movement: 내부 수평 이동(침해대응 문제 단골)

## 6) SSL 취약점

- Heartbleed/POODLE/FREAK/Logjam/DROWN 단골

# ✅ “시험 직전 체크” 

1. utmp=현재, wtmp=이력, btmp=실패
2. 바이러스도 자가증식 O / 웜은 전파 O / 트로이는 위장
3. DNS 포트 53, FTP 제어 21, TFTP 69/UDP, SFTP 22
4. IPv4 멀티캐스트 Class D
5. traceroute = TTL 증가 + ICMP Time Exceeded
6. CBC는 IV 필수 / Padding oracle
7. CTR은 비트오류 국소적
8. IDS는 탐지, IPS는 탐지+차단
9. EAL 숫자 클수록 높음
10. inode에 파일명 없음 / 하드링크 삭제해도 남음
