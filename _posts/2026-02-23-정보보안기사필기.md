# ✅ Ch1. 시스템 보안

## 1) /etc/passwd 구성 — **계패유지설홈쉘**
* **구성 요소**: 계정, 패스워드 표시(x), UID, GID, 설명, 홈 디렉터리, 로그인 쉘.
* **패스워드 보호**: 실제 해시값은 `/etc/shadow`에 저장됨.
* **사용자 식별**: root의 UID는 0으로 고정됨.
* **프로그램**: `/usr/bin/passwd`는 비밀번호 변경을 위한 실행 프로그램임.

## 2) /etc/shadow 구성 — **유엔마민맥 경인만**
* **구성 요소**: 유저 계정, 암호화 패스워드, 마지막 변경일(1970년 기준 일수), 최소/최대 사용 기간, 경고 기간, 비활성 기간, 만료일.

## 3) 파일 타입, 권한 및 명령
* **파일 타입**: 일반 파일(`-`), 디렉터리(`d`), 심볼릭 링크(`l`).
* **권한 체계**: r(4), w(2), x(1) 체계 및 `chmod`, `chown`, `chgrp` 명령 활용.
* **umask (기출 단골)**: 기본 권한에서 마스킹 값을 뺀 실제 권한 설정.
    * 기본 파일: 666 / 디렉터리: 777.
    * **예시**: umask 027 적용 시 파일 640, 디렉터리 750 설정.

## 4) inode 및 링크
* **inode 특징**: 파일의 메타데이터를 담고 있으나 **파일명은 포함되지 않음**.
* **하드 링크**: 같은 inode 공유 및 수정 내용 반영. 원본 파일명을 삭제해도 링크가 존재하면 데이터 유지.
* **심볼릭 링크**: 경로 참조 방식. 원본 파일 삭제 시 연결이 끊어짐(깨짐).

## 5) find 명령을 통한 권한 탐색
* **SetUID 찾기**: `find / -perm -4000 -type f` 명령 사용.
* **권한 검색 옵션**:
    * `-perm -222`: 명시된 모든 비트가 설정된 경우(AND).
    * `-perm /222`: 명시된 비트 중 하나라도 설정된 경우(OR).

## 6) crontab 구성 — **분시일월요(명령)**
* **시간 설정**: `0 0 1 * * cmd` 형식 사용 시 매달 1일 0시 실행.
* **접근 제어**: `allow`, `deny` 파일을 통한 사용자 제한.

## 7) 특수 권한 — **특유그기(4/2/1)**
* **SetUID(4000)**: 실행 시 파일 소유자의 권한으로 실행됨.
* **SetGID(2000)**: 실행 시 그룹의 권한으로 실행되며, 디렉터리 적용 시 그룹 소유권 상속.
* **Sticky Bit(1000)**: 디렉터리 내 파일 삭제/이름 변경 권한을 소유자 및 root로 제한 (예: `/tmp`).

## 8) xinetd 및 TCP Wrapper
* **xinetd**: 서비스 활성화 여부, 소켓 타입, 동시 접속 제한(cps, instances), 접근 시간(access_time) 등 설정.
* **TCP Wrapper**: `/etc/hosts.allow` 및 `/etc/hosts.deny`를 통한 네트워크 접근 제어.

## 9) PAM (Pluggable Authentication Modules) — **어오패세**
* **모듈 유형**: auth(인증), account(정책), password(변경), session(세션).
* **Control Flag**: requisite, required, sufficient, optional 제공.

## 10) 시스템 로그
* **로그 관리**: 중앙 수집 및 원격 백업이 보안상 바람직함.

| 로그 종류 | 내용 및 확인 명령어 |
| :--- | :--- |
| **utmp** | 현재 로그인 중인 사용자 정보 (`who`) |
| **wtmp** | 로그인/로그아웃 및 부팅 이력 (`last`) |
| **btmp** | 로그인 실패 이력 (`lastb`) |
| **lastlog** | 사용자별 마지막 로그인 기록 |
| **sulog** | su(사용자 전환) 시도 기록 |
| **xferlog** | FTP 전송 로그 |

## 11) 패스워드 정책 및 세션 관리
* **정책 설정**: 최소/최대 사용 기간, 최소 길이 등 설정.
* **명령어**: `pwconv`(Shadow 전환), `pwunconv`(Shadow 해제).
* **TMOUT**: 일정 시간 입력 없을 시 쉘 자동 로그아웃 설정.

## 12) 프로세스 관리
* **좀비 프로세스**: 상태 코드 `Z`로 표시됨. `ps aux | grep Z` 등으로 확인 가능.

## 13) Windows 그룹 및 공유
* **기본 그룹**: Administrators, Account Operators, Backup Operators, Power Users(관리자 상위 아님).
* **기본 공유**: IPC$, ADMIN$, C$ 등 관리용 공유 자원.

# ✅ Ch2. 네트워크 보안

## 1) TCP/IP 계층 구조 (정상 용어 기준)
* **구성**: 응용 / 전송 / 인터넷 / **네트워크 접속(Network Access)** 계층.
* **주의**: 각 계층별 프로토콜(TCP, UDP, IP, ICMP, ARP) 배치 구분 필수.

## 2) ARP Spoofing (2계층)
* **원리**: IP 주소를 허위 MAC 주소로 매핑하여 통신 탈취.
* **대응**: 정적(Static) ARP 설정, ARP watch 사용, DHCP snooping 및 DAI 적용.

## 3) IP Spoofing (3계층)
* **대응**: Ingress/Egress 필터링(BCP38), uRPF 적용, ACL 설정.
* **관련 설정**: `rhosts`, `hosts.equiv` 파일의 권한을 600 이하로 엄격하게 제한.

## 4) ICMP Redirect (3계층)
* **공격**: 라우팅 테이블 변조를 통한 경로 가로채기.
* **방어**: `accept_redirects=0` 설정 및 `netstat -rn`을 통한 테이블 확인.

## 5) 주요 네트워크 명령
* **ping**: ICMP echo 요청/응답 활용.
* **traceroute**: TTL 값을 증가시키며 **ICMP Time Exceeded** 메시지를 유도하여 경로 추적.
* **promisc**: `ifconfig` 명령을 통한 무차별 모드 활성화로 패킷 스니핑 수행.

## 6) 포트 스캐닝 응답 패턴
| 방식 | Open 상태 응답 | Closed 상태 응답 |
| :--- | :--- | :--- |
| **TCP Connect** | SYN/ACK → ACK 응답 | RST 응답 |
| **SYN Scan (Half-open)** | SYN/ACK → RST 전송 | RST 응답 |
| **FIN / NULL / XMAS** | 무응답 | RST 응답 |
| **ACK Scan** | 방화벽 필터링 여부 확인용 | - |
| **UDP Scan** | 무응답 또는 응답 데이터 | ICMP Unreachable 응답 |

## 7) Nmap 옵션
* **스캔 옵션**: `-sS`(SYN), `-sT`(Connect), `-sU`(UDP), `-sA`(ACK).
* **기타**: `-sV`(버전), `-O`(운영체제), `-p`(포트), `-Pn`(Ping 생략), `-f`(단편화), `-D`(Decoy).
* **주의**: Connect 스캔(`-sT`)은 로그가 남으므로 스텔스 스캔이 아님.

## 8) DoS, DDoS 및 DRDoS
* **DoS**: 특정 서비스 반복 호출을 통한 자원 고갈 공격.
* **DRDoS**: 반사 서버(DNS, NTP 등 UDP 기반)를 이용한 반사 및 증폭 공격.
* **Smurf**: ICMP Directed Broadcast 활용. 방어 위해 `no ip directed-broadcast` 설정.

## 9) SYN Flood 대응 방안
* **기술적 대책**: SYN Cookies 사용, Backlog Queue 튜닝, 임계치(Rate Limit) 설정, 앞단 스크러빙 장비 운용.

## 10) VPN (IPSec 및 SSL)
* **IPSec (L3)**:
    * **AH**: 인증 및 무결성 제공 (암호화 미지원).
    * **ESP**: 암호화 필수, 인증/무결성 선택 지원.
* **비교**: IPSec은 패킷 단위, SSL VPN은 세션/메시지 단위 제어 수행.

## 11) 무선 보안
* **주소 체계**: IPv4 멀티캐스트는 Class D 대역 사용.
* **암호화 방식**: WPA(TKIP), WPA2(CCMP/AES). TKIP의 취약성 유의.

# ✅ Ch3. 응용 프로그램 보안

## 1) DNS (Domain Name System)
* **포트**: 일반 질의(53/UDP), 존 전송(53/TCP).
* **질의 방식**: 재귀적(Recursive) vs 반복적(Iterative) 구분 필요.
* **역조회**: `dig -x` 명령을 통한 PTR 레코드 확인.
* **보안**: DNSSEC 적용을 통한 스푸핑 및 캐시 포이즈닝 방지.

## 2) HTTP 및 쿠키 보안
* **보안 속성**: `HttpOnly`(스크립트 접근 차단), `Secure`(HTTPS만 전송), `SameSite`(CSRF 방어).

## 3) FTP 관련 프로토콜
* **FTP 포트**: 제어(21/TCP), 데이터(Active: 20/TCP, Passive: 1024 이후 임의 포트).
* **TFTP**: 69/UDP 사용하며 인증 기능이 없어 취약함.
* **전송 보안**: SFTP(SSH 기반 22/TCP), FTPS(SSL/TLS 기반).
* **FTP Bounce**: PORT 명령을 악용하여 제3자에게 패킷을 전송하게 만드는 공격.

## 4) SNMP (Simple Network Management Protocol)
* **포트**: 161/UDP(에이전트), 162/UDP(트랩).
* **버전**: v3 사용 권장(인증 및 암호화 지원). 커뮤니티 스트링 기본값 변경 필수.

## 5) 주요 웹 취약점 (SQLi, XSS, CSRF)
* **SQL Injection**: `1=1` 등 논리식 삽입. **Prepared Statement**로 대응.
* **XSS**: 스크립트 실행 공격. 출력 인코딩 및 CSP 적용으로 대응.
* **CSRF**: 사용자 의도와 무관한 요청 강제. 토큰 사용 및 Referer 확인으로 대응. (HTML 인코딩은 오답임)

## 6) 업로드, 다운로드 및 경로 보안
* **파일 업로드**: 확장자 검증, 저장 경로 격리, 실행 권한 제거.
* **디렉터리 리스팅**: `Options -Indexes` 설정을 통해 목록 노출 차단.

## 7) 이메일 보안
* **PGP vs S/MIME**: S/MIME은 X.509 인증서 체계 기반으로 동작.
* **포트**: SMTP(25, 465, 587), POP3(110, 995), IMAP(143, 993).

# ✅ Ch4. 보안 장비 운용

## 1) 주요 보안 솔루션
* **IDS**: 탐지 및 경보 시스템.
* **IPS**: 탐지 후 실시간 차단(In-line 구성) 수행.
* **WAF**: 웹 애플리케이션 계층(L7) 특화 차단.
* **UTM**: 방화벽, IPS, VPN 등 다수 기능을 통합한 장비.
* **NAC**: 단말의 네트워크 접속 제어 및 무결성 점검.

## 2) Snort 룰 구조
* **Action**: alert, log, drop, reject 등.
* **Option**: msg, sid, content, distance, within 등 활용.
* **임계치 설정**: `threshold`의 limit, both, threshold 옵션 구분 필수.

## 3) iptables 설정
* **체인(Chain)**: INPUT, OUTPUT, FORWARD.
* **정책**: ACCEPT, DROP, REJECT, LOG.
* **상태 매칭**: ESTABLISHED, RELATED 등 세션 상태 기반 필터링.

## 4) 무결성 및 취약점 점검 도구
* **무결성**: Tripwire.
* **루트킷**: chkrootkit.
* **취약점 스캐너**: Nessus, SAINT, SATAN.

## 5) APT 및 사이버 킬체인 (Kill Chain)
* **7단계**: 정찰 → 무기화 → 전달 → 취약점 공격 → 설치 → C&C → 목적 실행.
* **내부 이동**: 침투 후 권한 상승 및 **수평 이동(Lateral Movement)** 수행.

## 6) 주요 SSL/TLS 취약점
* **사례**: Heartbleed, POODLE, FREAK, Logjam, DROWN 등 명칭 숙지.

# ✅ Ch5. 정보보호 일반 및 관리

## 1) 보안의 3요소 (CIA)
* **기밀성, 무결성, 가용성**: 변경·삭제·생성 방지는 무결성에 해당함.

## 2) 접근 통제 모델
* **정책**: MAC(강제적), DAC(임의적), RBAC(역할 기반).
* **모델**:
    * **Bell-LaPadula (BLP)**: 기밀성 중시. No Read Up / No Write Down.
    * **Biba**: 무결성 중시.
    * **Clark-Wilson**: 상업적 용도, 무결성 강조.
    * **만리장성(Brewer-Nash)**: 이해 상충 방지.

## 3) 위험 처리 전략
* **회피**: 사업 중단 등으로 위험 원인 제거.
* **감소**: 보안 통제 적용으로 위험 수준 저하.
* **전가**: 보험 가입이나 외주를 통한 책임 이전.
* **수용**: 현재의 위험을 그대로 감내.

## 4) 위험 분석 기법
* **정량적**: 확률 분포, 수학 공식, ALE(연간 예상 손실액) 등 수치 활용.
* **정성적**: 델파이(전문가 합의), 시나리오, 순위 결정법 등 주관적 판단.

## 5) 디지털 포렌식
* **원칙**: 연계 보관성(Chain of Custody) 유지 필수.
* **아티팩트**: 웹 방문 이력, 쿠키, 캐시 등 증거 분석.

## 6) CC 인증 및 EAL 등급
* **EAL 1~7**: 등급 숫자가 높을수록 보증 수준이 높음을 의미함.

## 7) 공개키 암호 난제 구분
### 7-1) 대표 난제
* **소인수분해(Factoring)**: 큰 합성수 n을 소인수 p,q로 분해하는 문제의 어려움 기반.
* **이산대수(Discrete Logarithm)**: g^x ≡ y (mod p)에서 x를 찾기 어려움 기반.
* **ECC**: 타원곡선 이산대수(ECDLP) 기반.

### 7-2) 알고리즘/기술 매칭
| 기반 난제	| 대표 알고리즘/기술	| 포인트 |
| :--- | :--- | :--- |
| 소인수분해	| RSA, Rabin	| “RSA=Factoring” |
| 이산대수	| Diffie–Hellman, ElGamal, DSA	| “DH/ElGamal/DSA=DLP” |
| ECDLP	| ECC, ECDH, ECDSA	| “ECC 계열=ECDLP” |

### 7-3) 출제 포인트
* “이산대수가 아닌 것” 문제에서 보기 중 RSA가 정답으로 자주 출제됨.
* 키교환(DH/ECDH), 서명(DSA/ECDSA), PKI(RSA) 연계 출제 빈도 높음.

| 구분 | 목적 | 암호화/서명 시 사용 키 | 복호화/검증 시 사용 키 |
| :--- | :--- | :--- | :--- |
| **대칭키** | 기밀성 보장 | **비밀키 (Secret Key)** | **비밀키 (Secret Key)** |
| **비대칭키 (기밀성)** | 데이터 보호 | 수신자의 **공개키 (Public Key)** | 수신자의 **개인키 (Private Key)** |
| **비대칭키 (전자서명)** | 인증 및 부인방지 | 송신자의 **개인키 (Private Key)** | 송신자의 **공개키 (Public Key)** |

## 8) 해시 함수 스펙(블록/라운드/출력/체이닝)

| 알고리즘 | 설계자/계열 | 입력 블록 크기 | 라운드 수 | 출력(해시) 길이 | 내부 상태(체이닝 값) |
|---|---|---:|---:|---:|---:|
| MD4 | Rivest / MD 계열 | 512비트 | 48 (3×16) | 128비트 (16B) | 128비트 |
| MD5 | Rivest / MD4 기반 개선 | 512비트 | 64 (4×16) | 128비트 (16B) | 128비트 |
| SHA-1 | NIST/NSA / SHA 계열 | 512비트 | 80 | 160비트 (20B) | 160비트 |
| SHA-256 | SHA-2 | 512비트 | 64 | 256비트 (32B) | 256비트 |
| SHA-512 | SHA-2 | 1024비트 | 80 | 512비트 (64B) | 512비트 |

**시험 포인트(함정 방지)**
- **SHA-256은 블록 512비트**, **SHA-512는 블록 1024비트**로 출제되는 경우가 많음.
- 위 표 기준으로 **내부 상태(체이닝 값) 길이 = 출력 길이**(MD5 128, SHA-1 160, SHA-256 256, SHA-512 512).
- **MD5, SHA-1은 충돌 취약** → 실무 권장은 SHA-256 이상.

# ✅ Ch6. 정보보호 제도

## 1) ISMS 및 ISMS-P
* **ISMS-P**: 정보보호 관리체계에 개인정보 보호 요구사항이 통합된 인증 제도.

## 2) 개인정보 보호 조치
* **처리 방침**: 목적, 항목, 보유 기간, 위탁 및 제공 사항 포함 필수.
* **안전성 확보**: 접근 통제, 암호화, 접속 기록 보관, 물리적 보안 등 수행.

## 3) 데이터 분류
* **고유식별정보**: 주민번호, 여권번호, 운전면허번호, 외국인등록번호 4종 중심 관리.

# 💡 시험 직전 체크
1. **로그 확인**: utmp(현재), wtmp(이력), btmp(실패).
2. **악성코드 구분**: 바이러스(자가증식), 웜(네트워크 전파), 트로이 목마(정상 위장).
3. **주요 포트**: DNS(53), FTP(21), TFTP(69/UDP), SFTP(22).
4. **멀티캐스트**: IPv4 Class D 대역 사용.
5. **Traceroute**: TTL 증가와 **ICMP Time Exceeded** 응답 활용.
6. **암호 구조**: CBC는 초기화 벡터(IV) 필수.
7. **네트워크 모델**: EAL 등급은 숫자가 클수록 높음.
8. **inode 정보**: 파일명은 포함되지 않으며, 하드 링크 삭제 시에도 데이터가 남을 수 있음.
9. **공개키 난제**: RSA=소인수분해 / DH·DSA·ElGamal=이산대수 / ECC=ECDLP
