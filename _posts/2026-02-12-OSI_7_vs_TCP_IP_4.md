## 1. 네트워크 모델 개요
* **OSI 7계층**: 네트워크 통신 과정을 7단계로 표준화하여 설명한 이론적 모델.
* **TCP/IP 4계층**: OSI 모델의 복잡성을 단순화하여 실제 인터넷 환경에서 표준으로 사용하는 실무적 모델.
* **계층 간 상호작용**: 하위 계층은 상위 계층에 기능을 제공하고, 상위 계층은 하위 계층의 기능을 이용하는 구조.
* **구현 방식**: 일반적으로 상위 계층(애플리케이션 쪽)은 소프트웨어로, 하위 계층(네트워크 하드웨어 쪽)은 하드웨어로 구현됨.

## 2. 캡슐화(Encapsulation)와 역캡슐화(Decapsulation)
데이터 전송 시 각 계층을 거치며 정보가 추가되거나 제거되는 과정.

* **캡슐화(Encapsulation)**: 상위 계층에서 내려온 데이터에 해당 프로토콜의 특성을 담은 헤더(Header)를 부착하여 하위 계층으로 전달하는 절차.
* **역캡슐화(Decapsulation)**: 수신 측에서 하위 계층부터 상위 계층으로 올라가며 헤더 정보를 순차적으로 제거하여 원본 데이터를 얻는 과정.
* **물리적 전송**: 최종적으로 물리 계층에서 이진 데이터(Binary Data)로 변환되어 전기 신호 등을 통해 전송됨.

## 3. 계층 모델의 상세 구조 및 변화

<img src="/images/2026-02-12-OSI_7_vs_TCP_IP_4/003.png" style="max-width:100%;"  />


### 계층별 주요 프로토콜 매핑

| TCP/IP 계층 | OSI 7계층 | 관련 프로토콜 예시 |
| :--- | :--- | :--- |
| **Application** | Application, Presentation, Session | HTTP, SMTP, FTP, Telnet |
| **Transport** | Transport | TCP, UDP |
| **Internet** | Network | IP, ICMP, ARP |
| **Network Interface** | Data Link, Physical | Ethernet, Wi-Fi, MAC |


### 모델의 세분화 (4계층 → 5계층)
* **갱신 사유**: 네트워크 표준의 지속적인 갱신으로 인해 하위 레이어의 명확한 구분이 필요해짐.
* **변화 내용**: 기존 TCP/IP 4계층의 'Network Interface Layer'가 'Data Link Layer'와 'Physical Layer'로 분리되어 **TCP/IP 5계층 모델**로 정의됨.

## 💡 핵심요약

**Q. OSI 7계층과 TCP/IP 4계층의 결정적 차이는?**
* OSI 7계층은 통신 표준화를 위한 이론적 가이드라인이며, TCP/IP 4계층은 이를 단순화하여 실제 인터넷 통신에 최적화된 실무 표준임.
* OSI 모델은 계층이 세분화되어 있어 교육 및 이론 학습에 적합하고, TCP/IP 모델은 통합된 계층 구조로 효율적인 실무 처리에 유리함.

**Q. 데이터 전송 시 캡슐화가 중요한 이유는?**
* **정보 전달**: 각 계층의 프로토콜이 필요한 제어 정보(주소, 오류 제어 등)를 헤더에 담아 정확한 목적지까지 데이터가 도달하도록 보장하기 때문임.
* **계층 독립성**: 각 계층은 자신의 헤더 정보만 처리하면 되므로, 다른 계층의 세부 구현 사항을 알 필요 없이 독립적으로 작동할 수 있음.
