## 1. 파일 내용을 보는 방법

Linux에서 파일에 작성된 내용을 확인하는 방법은 여러 가지가 있다.
입문 단계에서는 먼저 아래 두 가지 방식만 익혀두면 된다.

| 방법 | 용도 |
|---|---|
| `vi` 또는 `vim` | 파일을 편집기로 열어서 내용 확인 |
| `cat` | 파일 내용을 터미널에 바로 출력 |

파일을 수정하면서 확인해야 한다면 `vim`을 쓰고, 짧은 파일 내용을 빠르게 확인하고 싶다면 `cat`을 쓰면 된다.

---

## 2. Vim으로 파일 내용 보기

`vim` 또는 `vi`로 파일을 열면 파일 내용을 편집기 화면에서 확인할 수 있다.

```bash
vi app.txt
```

파일이 이미 존재하면 기존 파일 내용이 열린다.
파일 내용을 수정하지 않고 보기만 할 수도 있고, 필요하면 Insert mode로 전환해서 수정할 수도 있다.

Vim 공식 사용자 매뉴얼은 Vim을 시작하면 Normal mode에 있으며, `i`를 누르면 Insert mode로 들어간다고 설명한다. [1]
따라서 보기만 할 때는 굳이 Insert mode로 들어가지 않아도 된다.

---

## 3. `cat`으로 파일 내용 보기

`cat` 명령어는 파일 내용을 터미널에 출력한다.

```bash
cat app.txt
```

GNU Coreutils 문서는 `cat`을 파일 내용을 이어 붙여 표준 출력으로 쓰는 명령어로 설명한다. [2]
입문 단계에서는 "파일 내용을 터미널에 출력하는 명령어"로 이해하면 충분하다.

짧은 설정 파일이나 간단한 텍스트 파일을 빠르게 확인할 때는 `cat`이 편하다.

---

## 4. 텍스트 양이 많은 파일 만들기

텍스트 양이 많은 파일을 직접 만들어서 `cat`과 `vim`의 차이를 확인해보자.

```bash
seq 1 10000 > dummy.txt
```

위 명령어는 1부터 10000까지의 숫자를 줄마다 출력해서 `dummy.txt` 파일에 저장한다.
지금은 `seq`와 `>`의 자세한 의미를 몰라도 된다.

생성된 파일은 아래 두 방식으로 확인할 수 있다.

```bash
cat dummy.txt
vi dummy.txt
```

`cat dummy.txt`는 파일 전체 내용을 터미널에 한 번에 출력한다.
파일이 크면 터미널에 많은 내용이 쏟아져서 확인하기 불편할 수 있다.

반면 `vi dummy.txt`는 편집기 안에서 파일을 열기 때문에 긴 파일을 위아래로 이동하며 확인하기 좋다.

---

## 5. `cat`과 `vim` 사용 기준

파일 내용의 양에 따라 도구를 다르게 선택하면 편하다.

| 상황 | 추천 명령어 | 이유 |
|---|---|---|
| 텍스트 양이 적은 파일 | `cat` | 터미널에서 빠르게 확인 가능 |
| 텍스트 양이 많은 파일 | `vi` 또는 `vim` | 화면 이동 기능으로 탐색하기 편함 |
| 파일 내용을 수정해야 하는 경우 | `vi` 또는 `vim` | 편집까지 가능 |

항상 `vim`만 사용해도 문제는 없다.
다만 짧은 파일을 빠르게 확인할 때는 `cat`이 더 간단하다.

---

## 6. Vim에서 파일 맨 끝으로 이동하기

긴 파일을 열었을 때 파일 맨 끝으로 이동하려면 Normal mode에서 `G`를 누른다.

```text
G
```

Vim motion 문서는 `G`가 파일의 마지막 줄로 이동하는 명령이라고 설명한다. [3]

Vim에서 `G`는 대문자다.
따라서 키보드에서는 `Shift + g`로 입력할 수 있다.

---

## 7. Vim에서 파일 맨 처음으로 이동하기

파일 맨 처음으로 이동하려면 Normal mode에서 `gg`를 입력한다.

```text
gg
```

Vim motion 문서는 `gg`가 파일의 첫 번째 줄로 이동하는 명령이라고 설명한다. [3]

긴 파일을 읽다가 처음으로 돌아가야 할 때 자주 사용한다.

---

## 8. Vim에서 한 페이지씩 이동하기

긴 파일을 방향키로 한 줄씩 이동하면 시간이 오래 걸린다.
이럴 때는 한 페이지 단위로 이동하는 단축키를 사용하면 된다.

| 단축키 | 의미 | 기억 방법 |
|---|---|---|
| `Ctrl + f` | 한 화면 아래로 이동 | forward |
| `Ctrl + b` | 한 화면 위로 이동 | backward |

Vim motion 문서는 `CTRL-F`를 한 화면 앞으로 스크롤하고, `CTRL-B`를 한 화면 뒤로 스크롤하는 명령으로 설명한다. [3]

긴 파일을 볼 때는 아래 흐름으로 사용하면 편하다.

1. `vi dummy.txt`로 파일을 연다.
2. `Ctrl + f`로 아래쪽 내용을 확인한다.
3. `Ctrl + b`로 위쪽 내용으로 돌아간다.
4. `G`로 파일 맨 끝으로 이동한다.
5. `gg`로 파일 맨 처음으로 돌아간다.

---

## 핵심 정리

- `vi app.txt`는 파일을 Vim으로 열어서 내용을 확인한다.
- `cat app.txt`는 파일 내용을 터미널에 바로 출력한다.
- 짧은 파일은 `cat`으로 빠르게 확인하기 좋다.
- 긴 파일은 `vim`으로 열어 이동 기능을 쓰는 편이 좋다.
- Vim에서 `G`는 파일 맨 끝으로 이동한다.
- Vim에서 `gg`는 파일 맨 처음으로 이동한다.
- Vim에서 `Ctrl + f`는 한 페이지 아래, `Ctrl + b`는 한 페이지 위로 이동한다.
