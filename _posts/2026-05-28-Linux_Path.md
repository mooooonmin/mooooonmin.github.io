## 1. 상대 경로와 절대 경로가 필요한 이유

`/var` 디렉터리 안에는 보통 `log` 디렉터리가 있다.
`/var`로 이동한 뒤 `log` 디렉터리로 들어간다고 생각해보자.

```bash
cd /var
ls
```

이 상태에서 아래 두 명령어는 다르게 동작한다.

```bash
cd /log
cd log
```

`cd /log`는 실패할 수 있지만, `cd log`는 정상적으로 `/var/log`로 이동할 수 있다.
두 명령어의 차이를 이해하려면 **절대 경로(absolute path)**와 **상대 경로(relative path)**를 구분해야 한다.

---

## 2. 절대 경로

절대 경로는 **루트 디렉터리(`/`)를 기준으로 파일이나 디렉터리 위치를 표현하는 경로**다.
Linux man-pages의 `path_resolution(7)` 문서는 `/` 문자로 시작하는 경로를 absolute pathname이라고 설명한다. [1]

절대 경로는 반드시 `/`로 시작한다.

```text
/var
/var/log
/home/ubuntu
```

예를 들어 아래 명령어는 루트 디렉터리(`/`) 아래에 있는 `var` 디렉터리로 이동한다.

```bash
cd /var
```

`cd /log`는 루트 디렉터리(`/`) 바로 아래에 있는 `log` 디렉터리로 이동하라는 뜻이다.
만약 루트 디렉터리 바로 아래에 `log` 디렉터리가 없다면 `No such file or directory` 오류가 발생한다.

---

## 3. 상대 경로

상대 경로는 **현재 디렉터리를 기준으로 파일이나 디렉터리 위치를 표현하는 경로**다.
Linux man-pages의 `path_resolution(7)` 문서는 `/` 문자로 시작하지 않는 경로를 relative pathname이라고 설명한다. [1]

상대 경로는 `/` 이외의 문자로 시작한다.

```text
log
./log
../log
```

예를 들어 현재 위치가 `/var`라면 아래 명령어는 현재 디렉터리 안에 있는 `log` 디렉터리로 이동한다.

```bash
cd log
```

따라서 현재 위치가 `/var`일 때 `cd log`의 최종 이동 경로는 `/var/log`다.

---

## 4. `.`과 `..`의 의미

상대 경로에서는 `.`과 `..`을 자주 사용한다.

| 표현 | 의미 |
|---|---|
| `.` | 현재 디렉터리 |
| `..` | 상위 디렉터리 |

POSIX `cd` 문서도 `.`과 `..`을 각각 현재 디렉터리와 부모 디렉터리를 나타내는 경로 구성 요소로 설명한다. [2]

아래 두 명령어는 현재 디렉터리에 있는 `log` 디렉터리로 이동한다는 점에서 같은 의미다.

```bash
cd log
cd ./log
```

반면 아래 명령어는 현재 디렉터리의 상위 디렉터리 안에 있는 `log` 디렉터리로 이동한다.

```bash
cd ../log
```

## 5. 상대 경로와 절대 경로 예시

아래 명령어들이 어떤 기준으로 경로를 해석하는지 비교해보자.

| 명령어 | 경로 종류 | 의미 |
|---|---|---|
| `cd /log` | 절대 경로 | 루트 디렉터리(`/`) 아래의 `log` 디렉터리로 이동 |
| `cd log` | 상대 경로 | 현재 디렉터리 아래의 `log` 디렉터리로 이동 |
| `cd ./log` | 상대 경로 | 현재 디렉터리(`.`) 아래의 `log` 디렉터리로 이동 |
| `cd ../log` | 상대 경로 | 상위 디렉터리(`..`) 아래의 `log` 디렉터리로 이동 |

실제로 `/var` 기준에서 테스트하면 차이를 더 쉽게 이해할 수 있다.

```bash
cd /var
ls

cd /log
cd log
cd ..
cd ./log
cd ../local
```

`cd /log`는 루트 디렉터리 아래의 `log`를 찾는다.
`cd log`와 `cd ./log`는 현재 디렉터리 아래의 `log`를 찾는다.
`cd ../local`은 현재 디렉터리의 상위 디렉터리 아래의 `local`을 찾는다.

## 6. 복잡한 상대 경로 해석하기

현업에서 아래처럼 복잡한 경로를 일부러 작성할 일은 거의 없다.
다만 연습용으로 해석해보면 상대 경로의 동작을 이해하는 데 도움이 된다.

```bash
cd /var/log/apt
cd .././../lib/./../../var
```

두 번째 명령어를 단계별로 해석하면 다음과 같다.

| 단계 | 경로 조각 | 현재 위치 |
|---|---|---|
| 시작 |  | `/var/log/apt` |
| 1 | `../` | `/var/log` |
| 2 | `./` | `/var/log` |
| 3 | `../` | `/var` |
| 4 | `lib/` | `/var/lib` |
| 5 | `./` | `/var/lib` |
| 6 | `../` | `/var` |
| 7 | `../` | `/` |
| 8 | `var` | `/var` |

최종 경로는 `/var`다.

---

## 정리

- 절대 경로는 루트 디렉터리(`/`)를 기준으로 위치를 표현한다.
- 절대 경로는 반드시 `/`로 시작한다.
- 상대 경로는 현재 디렉터리를 기준으로 위치를 표현한다.
- 상대 경로는 `/` 이외의 문자로 시작한다.
- `.`은 현재 디렉터리를 의미한다.
- `..`은 상위 디렉터리를 의미한다.
- `cd /log`와 `cd log`는 기준점이 다르기 때문에 서로 다른 경로를 찾는다.
