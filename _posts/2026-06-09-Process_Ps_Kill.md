## 1. 프로세스란?

프로세스(process)는 실행 중인 프로그램을 의미한다.

운영체제를 공부하기 전에는 프로그램(program)이라는 표현이 더 익숙할 수 있다.
하지만 Linux에서는 실행 중인 프로그램을 설명할 때 프로세스라는 표현을 자주 사용한다.

예를 들어 브라우저, 데이터베이스 서버, 웹 서버처럼 현재 실행 중인 대상은 모두 프로세스로 볼 수 있다.
프로세스는 운영체제가 관리하는 실행 단위이며, 각 프로세스에는 식별을 위한 PID가 부여된다.

PID는 Process ID의 줄임말이다.
프로세스를 조회하거나 종료할 때 이 PID를 사용한다.

---

## 2. 실행 중인 프로세스 조회하기

Windows에서 `Ctrl + Shift + Esc`를 누르면 작업 관리자를 열 수 있다.
작업 관리자에서는 현재 실행 중인 프로그램 목록을 확인할 수 있다.

Linux에서도 작업 관리자처럼 실행 중인 프로세스를 확인할 수 있는 명령어가 있다.
대표적으로 `ps` 명령어를 사용한다.

```bash
ps aux
```

`ps` 명령어는 현재 프로세스 상태를 보여준다.
Linux man-pages의 `ps(1)` 문서는 `ps`가 현재 프로세스들의 스냅샷을 보고한다고 설명한다. [1]

---

## 3. `ps aux` 출력 해석하기

`ps aux`를 실행하면 여러 줄이 출력된다.
입문 단계에서는 아래 항목을 먼저 보면 된다.

| 항목 | 의미 |
|---|---|
| `USER` | 프로세스를 실행한 사용자 |
| `PID` | 프로세스를 식별하기 위한 ID |
| `%CPU` | CPU 사용률 |
| `%MEM` | 메모리 사용률 |
| `COMMAND` | 프로세스를 실행할 때 사용한 명령어 |

예시 형태는 아래와 비슷하다.

```text
USER       PID %CPU %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND
root         1  0.0  0.1  22500  9000 ?        Ss   10:00   0:01 /sbin/init
ubuntu    1234  0.0  0.2  15000  5000 pts/0    S+   10:10   0:00 bash
```

여기서 프로세스를 종료할 때 가장 중요한 값은 `PID`이다.
`PID`를 알아야 `kill` 명령어로 특정 프로세스를 지정할 수 있다.

---

## 4. 특정 프로세스만 찾기

`ps aux`는 실행 중인 많은 프로세스를 한 번에 보여준다.
그래서 출력 결과가 길어지면 원하는 프로세스를 찾기 어렵다.

이때 `grep`을 함께 사용한다.

```bash
ps aux | grep amazon
```

이 명령어는 `ps aux` 출력 결과 중 `amazon`이라는 단어가 들어간 줄만 보여준다.

명령어 흐름은 아래와 같다.

| 단계 | 동작 |
|---|---|
| `ps aux` | 실행 중인 프로세스 목록을 출력한다. |
| `|` | 앞 명령어의 표준 출력을 뒤 명령어의 표준 입력으로 넘긴다. |
| `grep amazon` | `amazon`이 들어간 줄만 출력한다. |

예를 들어 `nginx` 프로세스를 찾고 싶다면 아래처럼 입력할 수 있다.

```bash
ps aux | grep nginx
```

실무에서는 전체 프로세스 목록을 그대로 읽기보다, `grep`으로 원하는 키워드만 필터링해서 보는 경우가 많다.

---

## 5. 프로세스 종료하기

프로세스를 종료할 때는 `kill` 명령어를 사용한다.

기본 형태는 아래와 같다.

```bash
kill [PID 값]
```

예를 들어 종료하려는 프로세스의 PID가 `1234`라면 아래처럼 입력한다.

```bash
kill 1234
```

Linux man-pages의 `kill(1)` 문서는 `kill`을 프로세스에 시그널을 보내는 명령어로 설명한다. [2]
`kill [PID]`를 실행하면 해당 PID의 프로세스에 종료 요청 시그널을 보낸다고 이해하면 된다.

---

## 6. 프로세스 조회 후 종료하는 흐름

프로세스를 종료할 때는 보통 아래 순서로 진행한다.

1. 실행 중인 프로세스 목록을 조회한다.
2. `grep`으로 원하는 프로세스를 찾는다.
3. 종료할 프로세스의 `PID`를 확인한다.
4. `kill [PID]`로 종료 요청을 보낸다.

예시는 아래와 같다.

```bash
ps aux | grep nginx
kill [PID 값]
```

실제 명령어에서는 `[PID 값]` 자리에 숫자를 넣는다.

```bash
kill 1234
```

프로세스가 정말 종료됐는지 다시 확인한다.

```bash
ps aux | grep nginx
```

---

## 7. 주의할 점

`kill`은 이름과 달리 항상 즉시 강제 종료를 수행하는 명령어가 아니다.
기본적으로는 프로세스에 종료 요청 시그널을 보내는 명령어이다.

그래서 프로세스가 종료 요청을 처리하는 데 시간이 걸릴 수 있다.
또한 권한이 없는 다른 사용자의 프로세스를 종료하려고 하면 실패할 수 있다.

처음에는 아래 원칙을 지키면 된다.

- 먼저 `ps aux | grep [키워드]`로 정확한 프로세스를 찾는다.
- 종료할 프로세스의 `PID`를 확인한다.
- 확실하지 않은 프로세스는 종료하지 않는다.
- 필요한 경우에만 `kill [PID]`를 사용한다.

## 정리

프로세스(process)는 실행 중인 프로그램을 의미한다.
Linux에서 실행 중인 프로세스를 확인할 때는 `ps aux`를 사용할 수 있다.

```bash
ps aux
```

출력 결과가 많을 때는 `grep`과 함께 사용한다.

```bash
ps aux | grep amazon
```

프로세스를 종료할 때는 `PID`를 확인한 뒤 `kill` 명령어를 사용한다.

```bash
kill [PID 값]
```

프로세스를 종료하기 전에는 반드시 어떤 프로세스인지 확인하고, 정확한 PID를 사용해야 한다.
