## 1. 권한이란?

Linux에서 권한(permission)은 파일이나 디렉터리에 누가 어떤 방식으로 접근할 수 있는지 정하는 기능이다.

Linux는 여러 사용자 계정을 만들어 사용할 수 있다.
따라서 특정 사용자가 다른 사용자의 파일을 마음대로 읽거나 수정하지 못하게 막아야 한다.
이때 사용하는 기준이 권한이다.

GNU Coreutils 문서는 각 파일에 사용자의 접근 종류를 제어하는 file mode bit가 있다고 설명한다. [1]
입문 단계에서는 이 file mode bit를 `r`, `w`, `x`로 표시되는 권한이라고 이해하면 된다.

---

## 2. 권한 확인 방법

파일이나 디렉터리의 권한은 `ls -l` 명령어로 확인할 수 있다.

```bash
cd /var/log
ls -l
```

GNU Coreutils 문서는 `ls -l`이 long format을 출력하며, 파일 유형, 파일 mode bit, 소유자 이름, 그룹 이름, 크기, 시간 정보 등을 보여준다고 설명한다. [2]

예를 들어 아래와 같은 출력이 있다고 가정해보자.

```text
-rw-r----- 1 syslog adm 1200 Jun  3 09:00 auth.log
drwxr-xr-x 2 root   root 4096 Jun  3 09:00 apt
```

첫 번째 줄을 나누어 보면 아래와 같다.

| 값 | 의미 |
|---|---|
| `-rw-r-----` | 파일 유형과 권한 |
| `syslog` | 소유자 |
| `adm` | 소유 그룹 |
| `1200` | 크기 |
| `auth.log` | 파일 이름 |

즉 `auth.log` 파일의 소유자는 `syslog`이고, 소유 그룹은 `adm`이다.

---

## 3. 소유자와 소유 그룹

Linux의 파일에는 소유자(owner)와 소유 그룹(group owner)이 있다.

Linux man-pages의 `inode(7)` 문서는 파일 inode에 파일 소유자의 사용자 ID와 파일 그룹 소유자의 그룹 ID가 기록된다고 설명한다. [3]
새로 생성된 파일의 사용자 ID는 생성한 프로세스의 effective user ID가 되고, 파일의 그룹 ID는 부모 디렉터리의 set-group-ID 설정 여부에 따라 부모 디렉터리의 그룹 ID 또는 생성한 프로세스의 effective group ID가 된다고 설명한다. [3]

입문 단계에서는 아래처럼 이해하면 된다.

| 항목 | 의미 |
|---|---|
| 소유자 | 파일이나 디렉터리를 소유한 사용자 |
| 소유 그룹 | 파일이나 디렉터리에 연결된 그룹 |

사용자가 파일을 만들면 보통 그 사용자가 소유자로 보인다.
다만 실제 소유자와 소유 그룹 결정은 프로세스의 effective ID와 부모 디렉터리 설정에 영향을 받을 수 있다. [3]

---

## 4. 기본 그룹 확인 방법

특정 사용자의 기본 그룹(primary group)을 확인하려면 `id` 명령어를 사용할 수 있다.

```bash
id 사용자명
```

예를 들어 `ubuntu` 사용자를 확인한다.

```bash
id ubuntu
```

출력 예시는 아래와 같다.

```text
uid=1000(ubuntu) gid=1000(ubuntu) groups=1000(ubuntu),4(adm),24(cdrom),27(sudo),30(dip),105(lxd)
```

GNU Coreutils 문서는 `id` 명령어가 지정한 사용자 또는 현재 프로세스의 사용자 ID, 그룹 ID, 보조 그룹 ID 정보를 출력한다고 설명한다. [4]

위 출력에서 `gid=1000(ubuntu)` 부분을 보자.
여기서 괄호 안의 `ubuntu`가 기본 그룹 이름이다.

| 출력 부분 | 의미 |
|---|---|
| `uid=1000(ubuntu)` | 사용자 ID와 사용자 이름 |
| `gid=1000(ubuntu)` | 기본 그룹 ID와 기본 그룹 이름 |
| `groups=...` | 사용자가 속한 그룹 목록 |

따라서 위 예시에서 `ubuntu` 사용자의 기본 그룹은 `ubuntu`이다.

## 5. 권한 10글자 읽는 방법

`ls -l` 출력의 맨 앞 값은 보통 10글자로 보인다.

```text
-rw-r-----
```

이 10글자는 아래처럼 나누어 읽는다.

| 위치 | 의미 |
|---|---|
| 1번째 글자 | 파일 유형 |
| 2~4번째 글자 | 소유자에게 허용된 권한 |
| 5~7번째 글자 | 소유 그룹에게 허용된 권한 |
| 8~10번째 글자 | 그 외 사용자에게 허용된 권한 |

예를 들어 아래 권한을 보자.

```text
-rw-r-----
```

나누면 아래와 같다.

| 부분 | 값 | 의미 |
|---|---|---|
| 파일 유형 | `-` | 일반 파일 |
| 소유자 권한 | `rw-` | 읽기, 쓰기 가능 |
| 소유 그룹 권한 | `r--` | 읽기 가능 |
| 그 외 사용자 권한 | `---` | 허용된 권한 없음 |

---

## 6. 1번째 글자: 파일 유형

권한 문자열의 1번째 글자는 파일 유형(file type)을 의미한다.

GNU Coreutils 문서는 `ls -l` 출력에서 파일 유형 문자를 설명한다. [2]
입문 단계에서 자주 보는 값은 아래 3개이다.

| 문자 | 의미 |
|---|---|
| `-` | 일반 파일 |
| `d` | 디렉터리 |
| `l` | 심볼릭 링크 |

예시는 아래와 같다.

```text
-rw-r----- 1 syslog adm  1200 Jun  3 09:00 auth.log
drwxr-xr-x 2 root   root 4096 Jun  3 09:00 apt
lrwxrwxrwx 1 root   root   21 Jun  3 09:00 link
```

각 줄의 첫 글자만 보면 파일 유형을 구분할 수 있다.

---

## 7. 2~4번째 글자: 소유자 권한

2~4번째 글자는 소유자에게 허용된 권한이다.

```text
-rw-r-----
 ^^^
```

일반 파일에서 `r`, `w`, `x`의 의미는 아래와 같다.

| 문자 | 이름 | 일반 파일에서 의미 |
|---|---|---|
| `r` | read | 파일 내용을 읽을 수 있음 |
| `w` | write | 파일 내용을 수정할 수 있음 |
| `x` | execute | 파일을 실행할 수 있음 |
| `-` | 없음 | 해당 권한이 없음 |

권한은 항상 `r`, `w`, `x` 순서로 표시된다.
해당 권한이 없으면 그 자리에 `-`가 표시된다.

예를 들어 `rw-`는 읽기와 쓰기는 가능하지만 실행 권한은 없다는 뜻이다.

---

## 8. 디렉터리에서 r, w, x의 의미

디렉터리에서도 `r`, `w`, `x`를 사용하지만 의미가 조금 다르다.

| 문자 | 디렉터리에서 의미 |
|---|---|
| `r` | 디렉터리 안의 이름 목록을 조회할 수 있음 |
| `w` | 디렉터리 안에 파일을 생성, 삭제, 이름 변경할 수 있음 |
| `x` | 디렉터리 안의 항목에 접근할 수 있음 |

GNU Coreutils 문서는 디렉터리의 execute 권한이 그 디렉터리 안의 파일에 접근할 수 있게 한다고 설명한다. [5]

따라서 디렉터리 내부로 `cd` 하거나 내부 파일을 실제로 다루려면 `x` 권한이 중요하다.
디렉터리에 `w` 권한이 있어도 `x` 권한이 없으면 내부 항목에 접근하는 작업이 제한될 수 있다.

---

## 9. 5~7번째 글자: 소유 그룹 권한

5~7번째 글자는 소유 그룹에게 허용된 권한이다.

```text
-rw-r-----
    ^^^
```

예를 들어 아래 권한에서 소유 그룹 권한은 `r--`이다.

```text
-rw-r-----
```

이 뜻은 소유 그룹에 속한 사용자는 파일을 읽을 수 있지만, 수정하거나 실행할 수는 없다는 의미이다.

---

## 10. 8~10번째 글자: 그 외 사용자 권한

8~10번째 글자는 그 외 사용자에게 허용된 권한이다.

```text
-rw-r-----
       ^^^
```

여기서 그 외 사용자란 소유자도 아니고, 소유 그룹에 속하지도 않은 사용자를 의미한다.

예를 들어 아래 권한에서 그 외 사용자 권한은 `---`이다.

```text
-rw-r-----
```

이 뜻은 그 외 사용자에게 읽기, 쓰기, 실행 권한이 모두 없다는 의미이다.

---

## 11. 예시로 권한 해석하기

아래 값을 직접 해석해보자.

```text
drwxr-x---
```

10글자를 나누면 아래와 같다.

| 부분 | 값 | 의미 |
|---|---|---|
| 파일 유형 | `d` | 디렉터리 |
| 소유자 권한 | `rwx` | 조회, 생성/삭제, 접근 가능 |
| 소유 그룹 권한 | `r-x` | 조회와 접근 가능, 생성/삭제 불가 |
| 그 외 사용자 권한 | `---` | 허용된 권한 없음 |

즉 이 디렉터리는 소유자는 자유롭게 사용할 수 있고, 소유 그룹은 들어가서 조회할 수 있지만 파일 생성이나 삭제는 할 수 없다.
그 외 사용자는 접근할 수 없다.

---

## 핵심 정리

- 권한(permission)은 파일이나 디렉터리에 누가 어떤 방식으로 접근할 수 있는지 정하는 기능이다.
- `ls -l`로 파일 유형, 권한, 소유자, 소유 그룹을 확인할 수 있다.
- `id 사용자명`으로 사용자의 기본 그룹과 소속 그룹을 확인할 수 있다.
- 권한 문자열의 1번째 글자는 파일 유형을 의미한다.
- 권한 문자열의 2~4번째 글자는 소유자 권한이다.
- 권한 문자열의 5~7번째 글자는 소유 그룹 권한이다.
- 권한 문자열의 8~10번째 글자는 그 외 사용자 권한이다.
- 일반 파일에서 `r`은 읽기, `w`는 쓰기, `x`는 실행을 의미한다.
- 디렉터리에서 `r`은 목록 조회, `w`는 생성/삭제/이름 변경, `x`는 내부 항목 접근을 의미한다.
