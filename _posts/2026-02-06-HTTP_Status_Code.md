## 1. HTTP Status Code
* 클라이언트 요청을 서버가 어떻게 처리했는지 알려주는 숫자값.
* 서버가 상태 코드를 잘 보내줘야 클라이언트가 그에 맞춰서 다음 동작을 결정할 수 있음.
* 100번대부터 500번대까지 분류되어 있음.

---

## 2. 상태 코드 상세 (1xx ~ 5xx)

### 1xx (Informational)
* 요청 수신되어 처리 중이라는 뜻인데, 거의 안 씀.

### 2xx (Successful) - 요청 정상 처리
| 코드 | 이름 | 설명 |
| :--- | :--- | :--- |
| `200` | **OK** | 요청 성공. 주로 GET 조회 후 데이터 줄 때 씀. |
| `201` | **Created** | POST 요청으로 리소스가 새로 만들어졌을 때임. |
| `204` | **No Content** | 요청은 성공했는데, 응답 바디에 보낼 데이터가 없을 때 씀. |

### 3xx (Redirection) - 추가 행동 필요
주로 다른 페이지로 이동(리다이렉팅)할 때 사용함.

| 코드 | 이름 | 설명 |
| :--- | :--- | :--- |
| `301` | **Moved Permanently** | 주소가 영구 이동함. 브라우저가 새 주소를 기억함. |
| `302` | **Found** | 주소가 일시 이동함. |
| `304` | **Not Modified** | 캐시 데이터 그대로 쓰라는 뜻. 데이터 본문 전송 안 함. |
| `307` | **Temporary Redirect** | 302와 비슷하나, 리다이렉트 시 메서드 유지가 필수. |
| `308` | **Permanent Redirect** | 301과 비슷하나, 리다이렉트 시 메서드 유지가 필수. |

> **💡 참고: PRG(Post/Redirect/Get) 패턴**
> POST 주문 후 새로고침 시 중복 주문을 막으려고 302 응답을 줘서 결과 조회(GET) 페이지로 보내버리는 방식.

---

### 4xx (Client Error) - 클라이언트 잘못
보내는 쪽(클라이언트)에서 주소를 잘못 썼거나 권한이 없는 경우.

| 코드 | 이름 | 설명 |
| :--- | :--- | :--- |
| `400` | **Bad Request** | 요청 구문이 틀렸거나 파라미터가 잘못된 경우. |
| `401` | **Unauthorized** | 인증 안 됨. 즉, 로그인이 필요한데 안 한 경우. |
| `403` | **Forbidden** | 로그인은 했는데 해당 데이터 볼 권한이 없는 경우. |
| `404` | **Not Found** | 요청한 리소스가 서버에 없음. 주소 틀렸을 때 자주 봄. |

---

### 5xx (Server Error) - 서버 잘못
서버 쪽 로직이 터졌거나 서버가 죽었을 때 발생.

| 코드 | 이름 | 설명 |
| :--- | :--- | :--- |
| `500` | **Internal Server Error** | 서버 내부 예외 발생 시 나오는 가장 일반적인 기본 코드. |
| `502` | **Bad Gateway** | 게이트웨이가 뒷단 서버로부터 잘못된 응답을 받았을 때. |
| `503` | **Service Unavailable** | 서버 과부하 또는 점검 중이라 일시적으로 서비스 불가. |
| `504` | **Gateway Timeout** | 뒷단 서버가 너무 늦게 응답해서 타임아웃 걸린 경우. |

---

> **💡 핵심 요약**
> * `2xx`: 성공함.
> * `4xx`: 클라이언트가 고쳐야 함.
> * `5xx`: 서버가 고쳐야 함.
