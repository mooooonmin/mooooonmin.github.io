## 1. 표준 출력이란?

리눅스에서 `ls` 명령어를 입력하면 실행 결과가 터미널 화면에 출력된다.

```bash
ls
```

겉으로 보기에는 `ls` 명령어가 터미널 화면에 직접 결과를 출력하는 것처럼 보인다.
하지만 내부적으로는 명령어의 실행 결과가 표준 출력이라는 통로를 통해 출력된다.

표준 출력(stdout)은 명령어의 정상 실행 결과를 출력할 곳으로 보내는 기본 출력 통로다.
표준 출력의 기본 목적지는 터미널 화면이다.

Ubuntu manpage는 표준 출력(stdout)을 표준 I/O 스트림 중 하나로 설명하고, 표준 출력은 output stream이라고 설명한다. [1]

---

## 2. 출력 함수는 화면에 직접 쓰는 것이 아니다

C 언어에서 `printf()`는 값을 출력할 때 사용하는 함수다.
Java의 `System.out.println()`, JavaScript의 `console.log()`와 비슷한 역할을 한다고 생각할 수 있다.

```c
printf("hello\n");
```

처음에는 `printf()`가 터미널 화면에 직접 글자를 찍는 것처럼 느껴진다.
하지만 정확히는 `printf()`가 출력 데이터를 표준 출력으로 보낸다고 이해하는 편이 좋다.

표준 출력의 기본 목적지가 터미널 화면이기 때문에, 사용자는 결과가 화면에 바로 출력된다고 느낀다.

---

## 3. 표준 출력의 기본 목적지는 터미널이다

명령어를 실행하면 정상 결과는 보통 터미널 화면에 표시된다.

```bash
pwd
```

예를 들어 현재 디렉터리가 `/home/ubuntu`라면 아래처럼 출력될 수 있다.

```text
/home/ubuntu
```

이때 `pwd` 명령어의 실행 결과는 표준 출력으로 전달된다.
그리고 표준 출력의 기본 목적지가 터미널 화면이기 때문에 결과가 화면에 보인다.

Linux man-pages의 `stdio` 문서는 프로그램 시작 시 표준 입력, 표준 출력, 표준 에러라는 세 개의 텍스트 스트림이 미리 열려 있다고 설명한다. [2]

---

## 4. 표준 출력은 다른 곳으로 보낼 수 있다

표준 출력의 기본 목적지는 터미널 화면이지만, 반드시 터미널에만 출력해야 하는 것은 아니다.
표준 출력의 목적지를 파일로 바꿀 수도 있다.

예를 들어 아래 명령어는 `ls` 결과를 터미널 화면에 출력하지 않고 `result.txt` 파일에 저장한다.

```bash
ls > result.txt
```

이렇게 `>` 기호를 사용하면 표준 출력의 목적지를 파일로 바꿀 수 있다.
이 작업을 출력 리다이렉션이라고 부른다.

명령어를 실행한 뒤 파일 내용을 확인해보면 `ls` 결과가 저장되어 있다.

```bash
cat result.txt
```

즉, 표준 출력은 단순히 화면 그 자체가 아니라 출력 결과가 이동하는 통로다.
그 통로의 목적지를 터미널 화면에서 파일로 바꿀 수 있다.

---

## 5. 표준 출력의 번호는 1이다

리눅스에서는 표준 입력, 표준 출력, 표준 에러에 번호가 있다.
이 번호를 파일 디스크립터라고 부른다.

| 이름 | 약어 | 파일 디스크립터 | 기본 목적지 |
|---|---|---|---|
| 표준 입력 | stdin | `0` | 키보드 입력 |
| 표준 출력 | stdout | `1` | 터미널 화면 |
| 표준 에러 | stderr | `2` | 터미널 화면 |

Ubuntu manpage는 프로그램 시작 시 `stdin`, `stdout`, `stderr`와 연결된 정수 파일 디스크립터가 각각 `0`, `1`, `2`라고 설명한다. [1]

그래서 아래 두 명령어는 표준 출력 관점에서 같은 의미로 볼 수 있다.

```bash
ls > result.txt
ls 1> result.txt
```

첫 번째 명령어는 `1`을 생략한 형태다.
기본적으로 `>`는 표준 출력인 파일 디스크립터 `1`을 파일로 보낸다.

---

## 6. 표준 출력과 표준 에러는 다르다

표준 출력은 명령어의 정상 결과를 출력하는 통로다.
반면 표준 에러(stderr)는 에러 메시지를 출력하는 통로다.

예를 들어 존재하지 않는 파일을 조회하면 정상 결과가 아니라 에러 메시지가 발생한다.

```bash
ls no-file
```

이때 출력되는 에러 메시지는 표준 출력이 아니라 표준 에러로 전달된다.
기본 목적지가 둘 다 터미널 화면이기 때문에 겉으로 보기에는 같은 화면에 출력되는 것처럼 보인다.

하지만 리다이렉션을 배울 때는 둘을 구분해야 한다.
정상 결과를 파일로 보내는 것과 에러 메시지를 파일로 보내는 것은 서로 다른 작업이기 때문이다.

## 핵심 정리

표준 출력(stdout)은 명령어의 정상 실행 결과를 출력할 곳으로 보내는 기본 출력 통로다.
표준 출력의 기본 목적지는 터미널 화면이다.

명령어가 결과를 화면에 직접 찍는 것처럼 보이지만, 실제로는 실행 결과가 표준 출력으로 전달되고 표준 출력의 목적지가 터미널 화면이기 때문에 화면에 보이는 것이다.

표준 출력은 파일로도 보낼 수 있다.

```bash
ls > result.txt
```

표준 출력의 파일 디스크립터 번호는 `1`이다.
그래서 `>`는 기본적으로 `1>`와 같은 의미로 사용할 수 있다.
