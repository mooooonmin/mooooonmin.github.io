## 1. 기본 작성 원칙

일반 포스트는 큰 흐름을 H2 제목으로 나누고, 세부 내용은 H3/H4와 목록, 표, 코드블록으로 정리한다.

- 자격증 포스트는 별도 형식을 유지한다.
- 일반 포스트의 큰 섹션은 H2만 사용한다.
- 하위 제목에는 번호를 붙이지 않는다.
- 제목 아래에는 빈 줄을 1줄 둔다.

---

## 2. 제목 규칙

큰 섹션 제목은 `## 1. 제목` 형식으로 작성한다.

```md
## 1. 큰 섹션 제목

본문 내용
```

하위 제목은 번호 없이 작성한다.

```md
### 하위 개념 제목

하위 내용
```

---

## 3. 본문과 목록 규칙

일반 설명은 하이픈 목록을 사용한다.

```md
- 첫 번째 설명
- 두 번째 설명
```

순서가 중요한 절차는 번호 목록을 사용한다.

```md
1. 첫 번째 단계
2. 두 번째 단계
3. 세 번째 단계
```

---

## 4. 표와 코드블록 규칙

표는 헤더, 구분선, 본문 행을 같은 구조로 작성한다.

```md
| 구분 | 설명 |
|---|---|
| 항목 | 설명 내용 |
```

명령어, 설정 파일, 출력 예시는 코드블록으로 분리한다.

````md
```bash
docker ps
```
````

---

## 5. 섹션 구분 규칙

큰 H2 섹션 사이에는 `---`를 사용한다.

```md
## 1. 첫 번째 섹션

본문

---

## 2. 두 번째 섹션

본문
```

H3/H4 사이에는 `---`를 넣지 않는다.

---

## 6. 요약과 출처 규칙

마지막 요약 섹션은 `## 핵심 정리`로 작성한다.

출처가 필요한 글은 가장 마지막에 `## 출처`를 둔다.

```md
## 핵심 정리

> 핵심 내용을 짧게 정리한다.

---

## 출처

1. 출처 이름
   URL
```

---

## 핵심 정리

> 일반 포스트는 H2에만 번호를 붙이고, H3/H4는 번호 없이 작성한다.
>
> 큰 섹션 사이는 `---`로 구분하고, 목록과 표는 정해진 Markdown 형식을 따른다.
>
> 자격증 포스트는 별도 형식을 유지한다.
